# QA·검토 게이트 (Reflection Gate) — #98

> 최종 갱신: 2026-06-09 (#98 도입)
> 마스터 헌장 §3의 실행 절차. 작성(도메인 페르소나)과 검토(QA·검토관)를 분리해 자가작성=권위를 차단한다.

## 1. 원칙
- **작성자 ≠ 검토자.** 회차에서 문서를 작성한 도메인 페르소나(MFDS/FDA/MDR/표준)와, 그것을 검증하는 QA·검토관은 분리된 판단으로 동작한다.
- 검토 미통과 산출물은 **커밋 금지**(반려 후 재작업).

## 2. 검토 체크리스트 (QA·검토관 수행)
1. **근거**: 1차 공식 출처 ≥3개, URL 유효, 발효·개정일 명시.
2. **최신성**: 2024~현재 기준 최신 개정 반영(QMSR 2026-02-02·EUDAMED 2026-05-28 등 기준선 대조).
3. **구조**: Fact/Interpretation/Action 3계층, 3지역×3제품 매트릭스(해당 시).
4. **정합성**: 기존 KB와 상충·중복 없음(INDEX 대조).
5. **추정 표기**: 미확정은 `[검증 필요]`로 명시 + VERIFICATION_REGISTER 등록.

## 3. 판정·기록
각 신규·갱신 문서 하단에 검토 라인 삽입:
```
---
작성: <페르소나> · 검토(QA): <근거 재확인 요약> · 상태: [승인|조건부|반려] · 일자: YYYY-MM-DD
```
- **승인**: 5개 항목 충족.
- **조건부**: 사람 최종 사인오프 필요(예: 회사 내부수치·전략판단) → `조건부 — 사람확인 대기`.
- **반려**: 미충족 항목 명시 후 재작업.

## 4. KPI 연동
- 근거 정확도(=승인 중 근거충족률) 100% 목표, 검토 반려율·재작업 건수 STATE_maintenance에 기록.
