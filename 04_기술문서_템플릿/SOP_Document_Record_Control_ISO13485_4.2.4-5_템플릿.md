**문서번호**: SOP-QMS-001 / SOP-QMS-002  
**버전**: 1.0  
**적용 표준**: ISO 13485:2016 §4.2.4 / §4.2.5 | QMSR 21 CFR §820.40 / §820.180 | MDR Annex IX | KGMP 고시 제2025-22호 §4.2.4/4.2.5

## Part A: Document Control (문서 관리) — SOP-QMS-001

### A-1. 목적 및 범위

QMS 운영에 필요한 모든 문서(절차서·작업지시서·양식·외부 문서 등)의 작성, 검토, 승인, 배포, 변경, 폐기를 통제하여 최신 승인본만 사용되도록 보장한다.

### A-2. 문서 유형 분류

| 계층 | 유형 | 예시 |
|---|---|---|
| Level 1 | Quality Manual | QM-001 |
| Level 2 | 절차서 (SOP) | SOP-DEV-001, SOP-QMS-001 등 |
| Level 3 | 작업지시서 (WI) / 방법서 | WI-PRD-001 |
| Level 4 | 양식 (Form) / 템플릿 | FM-DEV-001 |
| Level 5 | 외부 문서 | ISO 13485, IEC 60601-1, 규제 가이던스 등 |

### A-3. 문서 번호 체계

```
[유형]-[기능코드]-[일련번호]  예: SOP-DEV-001, WI-QMS-003, FM-RMF-002

기능코드:
  QMS: QMS 일반     DEV: 설계개발     MFG: 생산
  RMF: 위험관리     CST: 고객불만     CAP: CAPA
  SUP: 공급업체     AUD: 내부감사     MGT: 경영검토
  SFT: 안전시험     SW: 소프트웨어
```

### A-4. 문서 수명주기 절차

#### A-4.1 신규 문서 작성

1. 작성자는 해당 기능코드의 다음 일련번호로 문서 초안 작성.
2. 초안 상단에 `Status: DRAFT`, 버전 `0.1`로 표기.
3. 내용에 포함 필수:
   - 목적 / 범위 / 참조 문서 / 책임과 권한 / 절차 / 기록 / 개정 이력
4. 소관 부서장 검토 → QA Manager 최종 승인.
5. 승인 후 `Status: RELEASED`, 버전 `1.0`으로 갱신 후 배포.

#### A-4.2 문서 검토 및 개정

1. 정기 검토: 모든 문서는 **2년마다** 또는 규제 개정 시 적합성 검토.
2. 변경 필요 시 변경 사유를 `개정 이력` 란에 기록.
3. 변경된 내용은 식별 가능하도록 표시 (취소선 또는 변경 내역 기재).
4. 재승인 절차는 신규 작성과 동일.
5. 버전 규칙: 내용 변경 → 정수 증가 (1.0→2.0), 오탈자 수정 → 소수 증가 (1.0→1.1).

#### A-4.3 문서 배포 및 접근

1. 승인된 문서는 **eQMS** 또는 지정 공유 폴더에 게시.
2. 배포 매체:
   - 전자: eQMS 시스템의 최신본 자동 배포 (버전 잠금)
   - 종이: "통제사본" 스탬프 부착 + 배포 대장 기록
3. **비통제 사본**은 사용 금지. 외부 배포 시 "참조용" 표기 필수.
4. 외부 문서(표준, 규제)는 목록 관리하고 최신 판본 여부를 연 1회 확인.

#### A-4.4 문서 폐기

1. 구버전 문서는 모든 배포처에서 즉시 회수 또는 eQMS에서 비활성화.
2. 종이본은 파쇄 또는 "폐기" 스탬프 후 별도 보관 (오용 방지).
3. 전자본은 eQMS 내 `Archive` 폴더로 이동 — 검색에서 제외되나 기록 보존.
4. 폐기 기록: 문서 번호, 버전, 폐기 일자, 담당자 서명.

### A-5. 필수 기재 항목 (모든 문서 공통)

| 항목 | 위치 | 필수 여부 |
|---|---|---|
| 문서 번호 | 표지/헤더 | 필수 |
| 버전 / 개정 번호 | 표지/헤더 | 필수 |
| 제목 | 표지 | 필수 |
| 작성일 / 발효일 | 표지 | 필수 |
| 작성자 서명 | 표지 | 필수 |
| 검토자 서명 | 표지 | 필수 |
| 승인자 서명 | 표지 | 필수 |
| 개정 이력 표 | 말미 | 필수 |
| 적용 페이지 범위 | 헤더/푸터 | 권장 |

## Part B: Record Control (기록 관리) — SOP-QMS-002

### B-1. 목적 및 범위

QMS 운영 중 발생하는 모든 기록(품질 기록, 시험 결과, 교육 기록, 감사 기록 등)을 식별, 보존, 보호, 검색, 보존 기간 관리 및 폐기하는 방법을 규정한다.

### B-2. 기록과 문서의 구분

| 구분 | 정의 | 예시 |
|---|---|---|
| **문서** | 수행할 활동을 기술 (지시서) | SOP, WI, 도면 |
| **기록** | 수행된 활동의 증거 (결과물) | 시험 성적서, 검토 기록, 교육 이수증 |

### B-3. 기록 관리 요건

#### B-3.1 식별성
- 기록에 제품/프로세스 연결 정보, 날짜, 작성자 기재.
- DHF 기록: 해당 제품 유형 또는 제품군과 연계 식별.

#### B-3.2 가독성
- 모든 기록은 읽기 가능한 상태 유지.
- 전자 기록: 파일 형식 장기 접근 가능성 확인 (PDF/A 권장).
- 수기 기록: 영구잉크 사용, 수정 시 한 줄 취소선 + 서명 + 날짜 (화이트아웃 금지).

#### B-3.3 검색성
- 기록은 인덱스 또는 분류 체계에 따라 신속히 검색 가능해야 함.
- eQMS 사용 시: 문서 번호, 제품 코드, 날짜 기준 검색 기능 필수.

#### B-3.4 보호
- 물리적 보호: 방수·방화 캐비닛 또는 서버 이중화.
- 접근 통제: 역할 기반 접근 권한 (읽기/쓰기/수정 분리).
- 전자 기록: 정기 백업 + 무결성 확인 (checksum 또는 감사 로그).

### B-4. 보존 기간 기준

| 기록 유형 | MFDS (KGMP) | FDA (QMSR) | MDR 2017/745 |
|---|---|---|---|
| **DHF / 설계 이력** | 제조 중단 후 **15년** | 제조 종료 후 **2년** 이상 (§820.180) | 제조 후 **15년** 이상 (Art.10 §8) |
| **제조 기록 (DMR·DHR)** | 15년 | 2년 (§820.180) | 15년 |
| **품질 기록 (CAPA·불만·감사)** | 15년 | 2년 | 15년 |
| **임상 평가 기록** | 15년 | 제출 기록 영구 | 수명 이후 15년 |
| **교육 훈련 기록** | 재직 기간 + 3년 | 2년 | NB 요구 시 제공 |
| **내부 감사 기록** | 15년 | 2년 | 15년 |

> **적용 원칙**: 세 지역 중 가장 엄격한 기준(MFDS/MDR **15년**)을 일괄 적용한다.

### B-5. 기록 색인 (Record Index) 관리

기록 유형별 색인을 유지하여 보존 기간 만료 시 자동 알림 또는 정기 점검을 수행한다.

```
기록 색인 양식 (FM-QMS-010)
| 기록 ID | 기록 유형 | 관련 제품/프로세스 | 생성일 | 보존 만료일 | 보관 위치 | 담당자 |
|---------|---------|-------------------|--------|------------|----------|--------|
| REC-001 | DHF Index | X-ray Detector | 2026-01-10 | 2041-01-10 | eQMS/DHF | QA Mgr |
```

### B-6. 기록 폐기

1. 보존 기간 만료 확인 후 QA Manager 승인 하에 폐기.
2. 종이 기록: 파쇄 후 폐기 증명서 보관.
3. 전자 기록: 안전 삭제 (overwrite 또는 인증된 삭제 도구 사용).
4. 폐기 기록: 기록 ID, 폐기 일자, 승인자 서명 보관 (별도 폐기 대장).
