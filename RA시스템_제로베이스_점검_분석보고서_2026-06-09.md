# RA 시스템 제로베이스 점검·분석 보고서 — 2차 도약 설계

**목표**: 지구상에 존재하는 RA(의료기기 인허가) 최강 시스템 구축
**대상**: X-ray Detector · Handheld X-ray Source · 촬영실 GUI SW / MFDS·FDA·MDR

---

## 0. 총평 (결론 먼저)

1차는 **"정교한 자동 수집·커밋 머신 + 우수한 콘텐츠"**를 만들어냈습니다. 141개 문서, 3지역×3제품 일관 매핑, 제출급 템플릿, 실거래 심사 이력까지 — 콘텐츠 자산은 상위권입니다.

그러나 사용자의 직관이 정확합니다. **전략·우선순위·품질을 총괄하는 "마스터(리딩 전문가 페르소나)"가 설계상 존재하지 않습니다.** 현 시스템은 매 회차 기억 없이 깨어나 BACKLOG 큐를 기계적으로 처리하고 잠드는 구조라, "전문가가 지시하는 대로" 흐르지 못하고 큐가 시키는 대로만 자랍니다. 성장 속도가 느린 근본 원인이 이것입니다.

세계 최강 RA 시스템의 본질은 **단일 진실원천 + 변경 자동전파 + 선제적 규제 인텔리전스 + 에이전트 오케스트레이션 + KPI 거버넌스**인데, 현재 가장 크게 빠진 축이 바로 **오케스트레이션(마스터)과 거버넌스**입니다.

## 1. 현황 진단 — 1차로 구축된 것

규모: md 141개(약 2.4MB), 01~07 분류 체계, EP 168/168(빌드 100%), 주간 모니터링 + 분기 심층패치 자동화 가동.

**강점 (감사 확인):**
- 최신 규제 사실의 정확성 — 예: Q3 패치가 COM(2025)1023을 "제안·미채택"으로 정확히 구분, EUDAMED 4모듈 의무화(2026-05-28)·legacy 마감(2026-11-28) 정확.
- Fact → Interpretation → Action 3계층 서술이 실제 산출물에 일관 구현.
- 제출 즉시성 있는 템플릿 — 예: ISO 14971 위험관리 파일(419줄)은 플레이스홀더만 채우면 제출 가능 수준.
- 3지역×3제품 매트릭스가 누락 없이 반복 — 저장소 전체 설계 일관성 높음.
- 심사 QA 이력은 합성이 아닌 실제 이메일 스레드 추출물(신뢰 가능한 실거래 자산).

## 2. 제로베이스 결함·공백 (정밀 감사)

### 구조 위생
- **교육 폴더 중복(확정)**: `05_전문가_양성교육`(2파일, 구)와 `05_전문가교육`(13파일, 신)이 같은 번호로 공존. 일부 파일 바이트 단위 완전 동일 → 구 폴더 고아화.
- **STATE 이중 모순(중대)**: 빌드용 `STATE.md`는 아직 `missed_run:true / missed_ep 9`(CATCHUP 활성)로 남아 유지관리 완료 사실과 충돌. 권위 파일 `STATE_maintenance.md`와 상충하며 폐기 표시 없음.
- **스케줄러 잔재**: `.ra-scheduler/`에 빈 디렉터리 `PENDING_PUSH/`, `_dash_build.html`(대시보드 staging 사본) 미정리.
- **파일명 위생**: `06_심사_QA이력`에 괄호·`&`·`+`·공백 포함 파일명 → shell·동기화 스크립트 파싱 오류 실제 재현.
- **헤더 누락 5종**: `*_인허가_상세가이드`(MFDS/FDA/MDR)·`규제_프레임워크_요약`·`사이버보안_통합가이드`가 SOP 의무 헤더(최종갱신·근거 URL) 없음 — 신규자 진입문서가 가장 노후.

### 품질·거버넌스
- **`[검증 필요]` 33건 미해소**: 태그를 닫는 검증 워크플로 부재 → 초안이 권위 문서처럼 INDEX 등재.
- **peer review 0건**: "검토자"는 전부 템플릿 플레이스홀더. 작성=검증=커밋이 단일 에이전트 한 루프.
- **단일 권위 부재**: 어느 STATE가 truth인지 메타 표시 없음. "Single Source of Truth"를 자칭한 관리대장은 빈 표.
- **사람 주도 영역 정지**: `03_진행현안`(실 과제 0건)·`07_협력기관_컨택`(연락처 전부 공란) — 전략적 오너십 부재 정황.

## 4. 격차 분석 — 현재 vs 최강

| 특성 | 현 상태 | 격차 |
|---|---|---|
| 1 단일 진실원천 | △ KB는 있으나 STATE 갈라짐·진행현안 공백 | 정합성·완결성 부족 |
| 2 변경 자동전파 | ✗ 없음 | 신규 규제 변경 시 영향 문서 수동 식별 |
| 3 선제 인텔리전스 | △ 주간 모니터 도입(수집 단계) | 영향평가·우선순위 자동화 미흡 |
| 4 dossier 자동매핑 | ✗ 없음 | 제품 영향 연결 수동 |
| 5 AI작성+검토 | △ 작성 자동, **검토 분리 없음** | 품질 게이트 부재 |
| 6 오케스트레이션 | ✗ **없음(핵심 결여)** | 마스터·전문 페르소나 부재 |
| 7 KPI 거버넌스 | ✗ 없음 | 성과·품질 지표 없음 |
| 8 거버넌스/CRO | ✗ **없음(핵심 결여)** | 전략·우선순위 총괄 주체 없음 |
| 9 가시성·대시보드 | ○ 유지관리 대시보드 신설 | 양호(추적 KPI 확장 여지) |
| 10 RAG 그라운딩 | △ 근거 URL 기재 | RAG 검색·근거 자동연결 아님 |

**핵심 결여 2축 = ⑥ 오케스트레이션(마스터) + ⑧ 거버넌스(CRO/검토).** 이 둘이 사용자가 지적한 "리딩 마스터 부재"와 정확히 일치합니다.

## 5. 근본 원인 — 마스터 오케스트레이터 부재

1. 세 SOP 어디에도 **총괄 주체 정의 없음**. 회차는 "대화 기억 없음" 자기완결 스크립트 → 무상태 기계 실행.
2. 우선순위가 사람이 미리 짠 BACKLOG·EP 큐에 종속. 에이전트는 "최상단 처리"만, **무엇이 중요한지 재판단하지 않음**.
3. 작성=검증=커밋 단일 주체 → **품질을 반려·재작업 지시하는 상위 감독 없음**(`[검증 필요]` 33건 방치가 증거).
4. 마스터가 있었다면 먼저 챙겼을 진행현안·협력기관이 수개월 공란 → 전략 오너십 부재.

➡ 결론: 현재는 "정교한 자동 머신 + 정교한 규칙"이지, **"전문가 마스터가 총괄하는 지식·인허가 체계"가 아님.**

## 6. 권고 — 2차 설계: 마스터 + 전문가 페르소나 오케스트레이션

업계·학술 멀티에이전트 패턴(Orchestrator–Worker, Agents-as-Tools, Reflection 게이트)을 RA에 적용한 운영모델을 제안합니다.

### 6-1. 페르소나 체계 (7역할)
1. **마스터 — Chief RA Strategist (지구최강 RA 전략가 페르소나)**: 비전·우선순위·품질기준 설정, 회차 결과 검토·반려, BACKLOG 재우선순위화. 모든 스케줄러가 회차 시작 시 이 "마스터 헌장(charter)"을 로드해 *전략가의 지시대로* 움직임.
2. **규제 인텔리전스 에이전트**: horizon scanning(주간) → 영향평가 → 마스터에 우선순위 신호 emit.
3. **MFDS 전문가** / 4. **FDA(510k·QMSR) 전문가** / 5. **EU MDR 전문가** / 6. **국제표준(IEC·ISO) 전문가**: 도메인 심층 작성.
7. **QA·검토 에이전트(Reflection 게이트)**: 작성과 분리되어 정확성·근거·최신성 검증 후 승인/반려. 휴먼 사인오프 라인 부여.

### 6-2. 운영 모델
- **Orchestrator–Worker**: 마스터가 목표를 분해→도메인 전문가가 실행→QA 게이트 통과→커밋.
- **Agents-as-Tools**: 마스터가 필요 시 MFDS/FDA/MDR 전문가를 호출.
- **신호 기반 피드백**: 인텔리전스가 변경 탐지 시 마스터에 우선순위 신호.
- **결정론적 라우팅**: 재현성·추적성 위해 규칙 기반 라우팅 결합(규제 영역 특성).

### 6-3. 거버넌스 신설
- STATE 단일화 + KPI(감지 적시성·정확도·완전성·검증 폐쇄율) 대시보드 반영.
- `[검증 필요]` 폐쇄 워크플로 + 모든 자동 문서에 "검토·승인" 라인.
- 마스터 헌장이 단일 권위 기준.

## 7. 2차 로드맵 (단계별)

**Phase A — 위생·정합성 복구 (즉시, 저위험)**
STATE 단일화(구 STATE.md DEPRECATED 처리), 05 교육 폴더 중복 제거, `.ra-scheduler` 잔재 정리(_dash_build.html·PENDING_PUSH/), 헤더 누락 5종 소급+최신성 점검, 파일명 위생.

**Phase B — 마스터 페르소나 도입 (핵심)**
`MASTER_CHARTER.md`(Chief RA Strategist 헌장: 비전·우선순위 원칙·품질기준) + 전문 페르소나 사양 작성. 주간/분기 SOP가 회차 시작 시 헌장을 로드해 *전략가의 판단으로* BACKLOG를 재우선순위화하도록 개정.

**Phase C — 품질 게이트·KPI**
QA·검토 에이전트(작성/검토 분리) + Reflection 루프. KPI를 STATE·대시보드에 정의·표시. `[검증 필요]` 33건 일괄 정리.

**Phase D — 고도화 (최강 단계)**
변경→dossier 자동 매핑, RAG 그라운딩(근거 자동연결), 진행현안·협력기관 실데이터 채움(사람+에이전트 협업).

## 8. 다음 액션 (지시 대기)

권고 우선순위: **Phase A(위생 복구) 즉시 실행 → Phase B(마스터 헌장 작성)**. 이 둘만으로도 "전문가가 지시하는 대로 흐르는" 체계로 전환되어 성장 속도가 달라집니다.

승인해 주시면 (1) Phase A 정리·정합성 복구를 바로 실행하고, (2) 마스터 "Chief RA Strategist" 헌장 초안과 7-페르소나 사양을 작성해 스케줄러 SOP에 연동하겠습니다.
