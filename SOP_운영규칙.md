# RA 지식베이스 운영 SOP (Standard Operating Procedure)

> 본 SOP는 지식베이스(RA project)의 자료 수집·분류·활용 및 현안 처리 요청 프로세스를 정의한다.

## 1. 자료 수집·분류 프로세스

### 1.1 신규 자료 업로드
1. 지시자는 자료를 `99_원본자료_업로드저장소/`에 업로드한다.
2. 파일명 규칙: `[YYYYMMDD]_[출처]_[제목].확장자`
   - 예: `20260422_MFDS_의료용Xray기술문서작성가이드.pdf`
3. 업로드 후 지시자는 Claude에게 "분류 요청"을 지시한다.

### 1.2 분류 작업 (Claude 수행)
1. 원본 자료의 내용·출처·발행기관·유형을 확인한다.
2. 아래 분류 매트릭스에 따라 대상 폴더를 선정한다.
3. 원본은 해당 폴더의 `원본/` 하위에 이동·복사한다.
4. 별도 `.md` 요약본을 작성한다. 요약본 필수 항목:
   - 문서명·원문 식별번호·발행기관·발행일
   - 적용 대상 (제품군·지역)
   - 핵심 요구사항 요약 (불릿 5~15개)
   - 자사 제품 적용 포인트
   - 관련 표준·규제 cross-reference
5. `INDEX.md`에 링크를 추가한다.

### 1.3 분류 매트릭스 (요약)
| 자료 유형 | 목적지 |
|---|---|
| MFDS 고시·가이드라인 | `01_규제지식베이스/국내_MFDS/` |
| FDA 가이던스·CFR | `01_규제지식베이스/미국_FDA/` |
| MDR 원문·MDCG | `01_규제지식베이스/유럽_CE_MDR/` |
| IEC·ISO 표준 | `01_규제지식베이스/국제표준_IEC_ISO/` |
| 자사 기술문서·STED | `02_제품별_기술파일/` (제품별) |
| 심사 질의·답변서 | `06_심사_QA이력/` |
| 시험기관·NB·컨설팅 자료 | `07_협력기관_컨택/` |

## 2. 현안 처리 요청 프로세스

### 2.1 요청 형식 (지시자)
현안을 Claude에게 전달할 때 다음 정보를 함께 제공한다:
- **案件 ID** (기존 案件) 또는 **신규 여부**
- **제품명·모델**
- **요청 내용** (질의응답 / 문서 작성 / 전략 수립 / 일정 검토 등)
- **기한**
- **관련 첨부자료 위치**

### 2.2 처리 흐름 (Claude 수행)
1. 요청 내용을 파싱하여 案件을 식별한다.
2. `03_진행현안/案件_관리대장.md` 및 해당 案件 폴더를 참조한다.
3. 관련 규제·표준을 `01_규제지식베이스/`에서 검색한다.
4. 과거 유사 질의를 `06_심사_QA이력/`에서 검색한다.
5. 결과물을 해당 案件 폴더 또는 지시된 위치에 저장한다.
6. 답변은 **팩트 기반·간결**하게 요약하고, 근거 문서 경로를 명시한다.
7. 진행 상태 변화 시 관리대장을 갱신한다.

### 2.3 신규 案件 발생 시
1. `RA-YYYY-NNN` 형식으로 案件 ID 부여.
2. `03_진행현안/01_진행중/RA-YYYY-NNN_案件명/` 폴더 생성 (템플릿 복사).
3. `00_案件기본정보.md` 작성.
4. 관리대장에 등재.
5. 전체 일정표에 마일스톤 반영.

## 3. 품질 원칙
- **팩트 기반**: 규제 원문·공식 가이던스에 근거. 추정 내용은 명시적으로 "초안/확정 필요"로 표시.
- **근거 명시**: 모든 주장에 근거 문서 경로·조항을 병기.
- **최신성**: 규제 개정(예: QMSR, MDR 전환기간, 사이버보안 가이던스) 정기 점검.
- **일관성**: 문서 양식·명명규칙 준수.
