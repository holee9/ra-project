### 1-2. DQE 정의 및 의의

DQE(Detective Quantum Efficiency)는 X선 검출기가 입사 X선의 신호 대 잡음 비(SNR)를 디지털 출력 이미지로 얼마나 효율적으로 전달하는지 나타내는 핵심 성능 지표.

```
DQE(f) = SNR²_output(f) / SNR²_input(f)
       = [MTF(f)]² / [NNPS(f) × q̄]
```

| 파라미터 | 정의 | 단위 |
|---|---|---|
| `MTF(f)` | Pre-sampled Modulation Transfer Function | 무차원 (0~1) |
| `NNPS(f)` | Normalized Noise Power Spectrum | mm² |
| `q̄` | 입사 광자 플루언스 (Air Kerma × 변환 계수) | 광자/mm² |
| `f` | 공간 주파수 | lp/mm (line pairs per mm) |

## 4. Step-by-step 측정 절차

### Step 1. 검출기 Warm-up 및 초기화

1. 검출기를 사용 온도(통상 20±5°C)에서 최소 30분 안정화.
2. 제조자 권고 Calibration(Gain/Offset correction, Bad Pixel map) 적용.
3. Flat-field correction ON 상태에서 모든 측정 수행.

### Step 2. RQA 빔 품질 설정 및 검증

1. X선 발생장치를 목표 kV·Al 필터 조합 설정 (표 3-2 참조).
2. HVL 실측:
   - 전리함을 X선 빔 중심 축에 배치 (초점–검출기 거리 FDD = 150 cm 권고).
   - Al 흡수체를 순차 추가하며 선량 측정 → HVL 계산.
   - 허용 범위: 목표 HVL ±5%.
3. 빔 균일도(Uniformity): 검출기 면적 내 ±5% 이내 확인.

### Step 3. Lag/Ghosting 보정 프레임 획득 (IEC 62220-1-1:2015 신규)

IEC 62220-1-1:2015는 Lag(잔상) 및 Ghosting 영향을 DQE 측정 전 보정하도록 규정.

1. 대선량(2X_N) 사전 노출 → 검출기 충분한 Lag 유발.
2. Lag decay 측정 프레임 연속 획득 (제조자 readout 주기에 따라 10~30 프레임).
3. Lag 보정 계수 산출 → 이후 Flat-field 및 Edge 이미지에 적용.
4. Ghosting 시험: 사전 노출(2X_N) → 저선량(X_N/2) Flat-field 비교 → Ghosting 비율 정량화.

### Step 4. Flat-field 이미지 획득 (NNPS 측정용)

1. 균일 팬텀(PMMA 또는 Cu)을 검출기 전면에 밀착 배치.
2. 각 선량 레벨(X_N/2, X_N, 2X_N)에서 **최소 8프레임** 이상 획득.
3. 프레임 간 X선 OFF → 다음 노출 전 Lag 안정화 대기 (>3초 권고).
4. 이미지 저장: Raw(전처리 없음) 또는 Flat-field corrected (상태 기록 필수).

### Step 5. Edge 이미지 획득 (MTF 측정용)

1. 엣지 팬텀을 검출기 수평·수직 방향에 대해 **2~3° 기울기**로 배치.
   - 기울기 정확도: ±0.5° 이내 (기울기 오차 시 MTF 과대 추정 위험).
2. X_N 선량 조건에서 **최소 4프레임** 획득 (SNR 확보용 평균화).
3. Edge 위치: 검출기 중앙 및 주변부(최소 2개소) 측정 권고.
4. 수평(x방향) 및 수직(y방향) 별도 측정.

### Step 6. Air Kerma (입사 선량) 측정

1. 팬텀 제거 후, 동일 조건에서 전리함으로 Air Kerma 측정 (최소 5회 반복, CV ≤1%).
2. 역자승 법칙으로 검출기 입사면 Air Kerma 환산.
3. q̄ 계산:
   ```
   q̄ = Air Kerma [μGy] × 변환 계수 [광자·μGy⁻¹·mm⁻²]
   ```
   (변환 계수: RQA3=30,174, RQA5=29,653 — IEC 62220-1-1:2015 Annex A 값)

## 5. 데이터 처리 및 DQE 계산

### 5-1. Pre-sampled MTF 계산 (Edge method)

```
과정: Edge Image → Edge Profile 추출 → ESF(Edge Spread Function) → 
      미분 → LSF(Line Spread Function) → FFT → MTF
```

1. **ESF 추출**: 엣지에 수직인 라인별 픽셀 강도 추출 → 기울기 각도로 오버샘플링 ESF 구성.
2. **오버샘플링 인자**: 기울기 θ → 오버샘플링 비 ≈ 1/tan(θ). 통상 2~3°에서 약 20~30×.
3. **ESF 평균화**: IEC 62220-1-1:2015에서 최종 MTF는 **오버샘플링 ESF의 평균화**를 통해서만 산출 (2003판 대비 변경).
4. **LSF**: ESF를 수치 미분.
5. **FFT**: LSF의 절댓값 → 주파수 공간 변환 → f=0 에서 정규화 → MTF(f).
6. **Nyquist 주파수**: f_Nyq = 1 / (2 × pixel pitch [mm]).

### 5-2. NNPS (Normalized Noise Power Spectrum) 계산

```
과정: Flat-field Images → 평균 영상 차감 → 2D FFT → NPS → NNPS
```

1. **평균 영상**: N프레임 평균 → 배경 제거용.
2. **차분 영상**: 개별 프레임 − 평균 영상 → 고정 패턴 노이즈 제거.
3. **ROI 선택**: 64×64 또는 128×128 픽셀 ROI (겹치지 않게), 최소 32개 ROI 권고.
4. **2D NPS**:
   ```
   NPS(u,v) = [pixel_pitch²/(N_ROI × N_px²)] × |FFT(ROI)|²  평균
   ```
5. **1D NNPS**: 2D NPS의 행·열 방향 프로파일 추출 → 평균 픽셀값² 로 정규화.
   ```
   NNPS(f) = NPS(f) / mean_pixel_value²
   ```
6. 세 선량 레벨(X_N/2, X_N, 2X_N) 각각 계산 후 비교 → 선량 비례성 확인.

### 5-3. DQE 계산

```
DQE(f) = [MTF(f)]² / [NNPS(f) × q̄]
```

- `MTF(f)`: Step 5-1 결과 (X_N 조건에서 측정).
- `NNPS(f)`: Step 5-2 결과 (X_N/2, X_N, 2X_N 각각).
- `q̄`: Step 6 결과 (해당 선량 조건).

**보고 주파수 범위**: 0 ~ f_Nyquist (최소 0.5 lp/mm 간격 보고).

**필수 보고 포인트**:

| 보고 항목 | 의미 |
|---|---|
| `DQE(0)` | Zero-frequency DQE — f=0 외삽값, 검출기 절대 효율 지표 |
| `DQE(1 lp/mm)` | 1 lp/mm에서의 DQE |
| `DQE(f_Nyq/2)` | Nyquist 절반 주파수 |
| `DQE(f_Nyq)` | Nyquist 주파수 — 고주파 한계 효율 |

| **제출 지역** | FDA 510(k), MFDS 허가, MDR Technical Documentation 전 지역 |
| **주요 고려사항** | 픽셀 피치에 따른 Nyquist 주파수 결정 → 보고 주파수 범위 설정. 간접변환(CsI/Gd₂O₂S) vs 직접변환(a-Se) 형광체별 DQE 특성 상이 |
| **DHF 연계 문서** | 성능시험 성적서 → Technical File (Annex II §6.1) 또는 eSTAR Performance Testing 섹션 |
| **내부 사양 관리** | DQE(0) ≥ [제조자 규격값] 를 DHF 내 제품 사양서(Product Specification)에 선정의 |

### 7-2. Handheld X-ray Source

| 항목 | 내용 |
|---|---|
| **적용 필요성** | **간접 적용** — Handheld Source 자체 DQE 측정 대상 아님. 단, Source-Detector 시스템 시험 시 Detector 사양 DQE 참조 |
| **연계 고려** | Handheld Source의 빔 품질(HVL, kVp)이 IEC 62220-1-1 RQA 조건과 일치하는지 확인 필요 (RQA3/RQA5 범위 내 운용 설계 권고) |
| **시스템 시험** | Handheld Source + Detector 조합 시스템 성능 시험 시, Detector DQE 결과를 시스템 SNR 추정에 활용 |

### 7-3. 촬영실 GUI SW

| 항목 | 내용 |
|---|---|
| **적용 필요성** | **간접 적용** — SW 자체 DQE 측정 대상 아님. 단, SW가 이미지 처리(필터링, 강조) 수행 시 영향 |
| **SW 영향 주의** | DQE 측정은 **Raw(미처리) 또는 Flat-field corrected만 허용**. GUI SW의 이미지 처리(에지 강화, 노이즈 감소) 알고리즘 적용 전 Raw 이미지로 측정 필수 |
| **SRS 반영** | GUI SW SRS에 "DQE 측정용 Raw image export 기능" 요구사항 명시 권고 |
| **IEC 62304 연계** | SW 설계 변경이 이미지 처리 알고리즘에 영향 시 → Detector DQE 재시험 트리거 조건으로 Risk Management File에 기재 |
