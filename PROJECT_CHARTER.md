## 1. 프로젝트 개요

**명칭**: 의료용 X-ray System 인허가 전문가 지식베이스 (RA Knowledge Base)

**저장소**: https://github.com/holee9/ra-project

**대상 제품 3종**
- X-ray Detector (평판 검출기)
- 소형 X-ray Source (Handheld형)
- 촬영실 GUI 기반 SW (Medical Imaging Software)

**대상 지역 3곳**
- 국내: 식약처(MFDS)
- 미국: FDA
- 유럽: CE / MDR

**현재 단계**: 구축 완성 → 유지관리 단계
- P1~P5 전체 168 EP 완성 (2026-06-05 기준)
- 자동 유지관리 가동 중 (매일 2회 + 주간 모니터링 + 분기 심층 패치)

## 2. 목적

**1차 목적: 지식 자산화**
인허가 담당자 이직·교체 시에도 조직 내 규제 지식이 소멸되지 않도록 체계적으로 축적하고 유지한다.

**2차 목적: 담당자 역량 양성**
신규 인허가 담당자가 자사 3개 제품의 3개 지역 인허가를 독립 수행할 수 있도록 12주 교육 커리큘럼과 실습 자료를 제공한다.

**3차 목적: 실무 지원**
진행 중인 인허가 과제에 대해 규제 조회, 전략 수립, 기술문서 작성 지원을 즉시 제공한다.

**4차 목적: Regula 챗봇 지식원**
ra-med-bot(Regula)이 이 KB를 pgvector로 참조하여 실시간 규제 질의응답을 제공한다. 미답변 발생 시 KB 보강으로 이어지는 피드백 루프를 운영한다.

| L | 3 | 3개 지역 비교 + 3개 제품 맵핑 동반 대형 문서 |

### 4.3 Regula 연동 흐름

```
ra-med-bot 미답변 감지
    ↓
RA 담당자 → knowledge-gap 판단 → ra-project Issue 등록
    ↓
Cowork 스케줄링 자동 처리 (03:00 / 19:30)
    ↓
관련 문서 작성·보강 → 커밋 → push
    ↓
Regula pgvector 재ingestion → 이후 답변 가능
```

## 5. 품질 원칙

- **팩트 기반**: 규제 원문·공식 가이던스에 근거. 불확실 내용은 반드시 `[검증 필요]` 태그.
- **근거 명시**: 모든 주장에 근거 문서 경로·조항 병기.
- **최신성**: 주간 모니터링 + 분기 패치로 규제 개정 지속 반영.
- **3지역 관점 유지**: 특정 지역만 다루더라도 타 지역 영향 1줄 언급.
- **3제품 맵핑 포함**: 모든 문서에 자사 3개 제품 적용 관점 포함.
- **중복 금지**: 기존 문서와 내용 겹칠 경우 기존 문서 갱신이 원칙.
