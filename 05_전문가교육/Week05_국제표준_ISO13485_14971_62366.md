# Week 5: 국제표준 — ISO 13485 / ISO 14971 / IEC 62366

> **대상**: X-ray 인허가 담당자 (RA 전문가 교육 Week 5/12)
> **소요 시간**: 이론 3시간 + 실습 1.5시간 + 평가 30분

---

## 1. 학습 목표

이 모듈을 마치면 다음을 할 수 있어야 한다.

1. ISO 13485:2016, ISO 14971:2019, IEC 62366-1:2015+AMD1:2020 각각의 구조·발행 이력·핵심 요구사항을 설명한다.
2. 세 표준 간 입·출력 관계(Quality System → Risk Management → Usability)를 도식화한다.
3. MFDS·FDA·MDR 3개 지역에서 각 표준의 인정·요구 방식 차이를 비교한다.
4. 자사 3개 제품(X-ray Detector / Handheld X-ray Source / 촬영실 GUI SW)에 대한 표준 적용 범위와 Safety Class를 결정한다.
5. 각 표준의 핵심 산출물 목록을 작성하고 인허가 패키지에서 활용하는 방법을 설명한다.

## 2. 표준 개요 — 발행 이력·현황

### 2.1 ISO 13485:2016 — QMS (품질경영시스템)

| 항목 | 내용 |
|---|---|
| 발행 기관 | ISO/TC 210 |
| 현행 버전 | ISO 13485:2016 (2016-03-01 발행) |
| 개정 전 | ISO 13485:2003 (2016 발행으로 폐지) |
| 개정 준비 | ISO/DIS 13485:2024 진행 중 (2024년 기준 투표 진행) [검증 필요] |
| 페이지 수 | 본문 46페이지 + Annex |
| 핵심 목적 | 의료기기·서비스·부품 제조 조직의 QMS 요구사항 규정 |

**구조 (8개 섹션 + Annex)**

```
§1  적용범위
§2  인용 표준
§3  용어 및 정의
§4  품질경영시스템 (QMS 일반요구사항, 문서화 요구사항)
§5  경영 책임
§6  자원 관리
§7  제품 실현 (설계·개발, 구매, 생산, 검교정)
§8  측정·분석·개선 (내부감사, CAPA, 부적합 관리)

Annex A: 본 표준과 ISO 9001 상호관계
Annex B: 법규 요구사항 적용 지침
Annex ZA: EU MDR 2017/745 Annex IX와의 관계 (유럽판 EN ISO 13485)
```

**핵심 요구사항 (ISO 9001과의 차이점)**

- **§4.1.6**: 위험 기반 접근 (risk-based approach) — 위험관리 ISO 14971과 직접 연계
- **§6.4.2**: 작업 환경 (cleanroom 등 특수 환경 요구)
- **§7.3**: 설계 및 개발 (Design Controls) — FDA 21 CFR 820.30과 동등
- **§7.5.6**: 공정 밸리데이션 (멸균·소프트웨어 포함)
- **§7.5.9**: 추적성 (Implantable devices 및 Class III 필수)
- **§8.3**: 부적합 제품 관리 (불만·리콜 연계)

### 2.2 ISO 14971:2019 — 위험관리 (Risk Management)

| 항목 | 내용 |
|---|---|
| 발행 기관 | ISO/TC 210 |
| 현행 버전 | ISO 14971:2019 (2019-12-07 발행) |
| 개정 전 | ISO 14971:2007 (2019 발행으로 폐지) |
| 유럽 채택 버전 | EN ISO 14971:2019+A11:2021 (2021-10-06; ZA Annex로 MDR/IVDR 조화 명시) |
| 페이지 수 | 본문 34페이지 + Annex |
| 핵심 목적 | 의료기기 수명주기 전체에 걸친 위험관리 프로세스 요구사항 규정 |

**위험관리 7단계 프로세스 (Clause 4~9)**

```
[§4]  위험관리 계획 수립 (Risk Management Plan)
  ↓
[§5]  위험 식별 (Hazard Identification)
  ↓
[§6]  위험 추정 (Risk Estimation)
  ↓
[§7]  위험 평가 (Risk Evaluation) → 수용 기준 적용
  ↓
[§8]  위험 통제 (Risk Control) → 3단계: 안전 설계 → 방호 수단 → 정보 제공
  ↓
[§9]  잔류 위험 평가 (Residual Risk Evaluation & Overall B/R)
  ↓
[§10] 위험관리 파일 완성 + 생산후 정보 연결 (Production/Post-Production)
```

**2019 개정 주요 변경사항 (vs 2007)**

| 변경 항목 | 2007 | 2019 |
|---|---|---|
| 수용 기준 | ALARP 원칙 언급 | "넓게 수용 가능한(broadly acceptable)" 개념 도입 |
| 전체 잔류위험 | 개별 합산 방식 | 전체 benefit-risk 평가 필수 (§9) |
| 생산후 정보 | §9 에 통합 | §10 으로 분리·강화 |
| IEC TR 24971 | 가이던스 내장 | 별도 Technical Report(IEC TR 24971:2020)로 분리 |
| Annex C | 위험 추정 기법 | 개정·보완 |

### 2.3 IEC 62366-1:2015+AMD1:2020 — 사용적합성 공학 (Usability Engineering)

| 항목 | 내용 |
|---|---|
| 발행 기관 | IEC/SC 62A |
| 현행 버전 | IEC 62366-1:2015+AMD1:2020 (AMD1 발행: 2020-05-04) |
| 폐지 버전 | IEC 62366:2007 |
| 기술 보고서 | IEC TR 62366-2:2016 (Part 2: Guidance) |
| 핵심 목적 | 의료기기 사용자 인터페이스의 사용적합성 엔지니어링 프로세스 요구사항 규정 |

**프로세스 구조 (6단계)**

```
[§4]  사용적합성 엔지니어링 파일 계획
  ↓
[§5]  의도된 사용(Intended Use) 및 사용자 프로파일 정의
  ↓
[§6]  사용과 관련된 위험 식별 (Use-Related Hazard) ← ISO 14971 §5 연계
  ↓
[§7]  사용 명세서 (Specification of Use)
  ↓
[§8]  형성적 평가 (Formative Evaluation) — 반복 설계·검증
  ↓
[§9]  총괄적 평가 (Summative Evaluation) — 규제 제출용 최종 사용 시험
```

**AMD1:2020 주요 변경사항**

- 기존 "IFU의 사용" 관련 요구사항 명확화
- Use Error 정의 재정리 (의도치 않은 사용·예측 가능한 오용 구분)
- 형성적 평가 문서화 요구사항 강화

## 3. 표준 간 상호관계 (Golden Triangle)

세 표준은 의료기기 개발 수명주기에서 긴밀하게 연결된다.

```
┌─────────────────────────────────────────────────────┐
│           ISO 13485:2016 (QMS Framework)            │
│  §4.1.6 Risk-Based Approach ──────────────────────┐  │
│  §7.3 Design Controls ─────────────────────────┐  │  │
└──────────────────────────────────────────────┼──┼──┘
                                               │  │
           ┌───────────────────────────────────┘  │
           ↓                                       │
┌─────────────────────────┐   출력물 공유   ┌──────────────────────────┐
│  ISO 14971:2019          │ ←───────────→ │  IEC 62366-1:2015        │
│  (Risk Management)       │               │  (Usability Engineering) │
│                          │               │                          │
│  §5 Hazard Identification│ ←── Use Error │  §6 Use-Related Hazards  │
│  §7 Risk Evaluation      │               │  §8 Formative Evaluation │
│  §9 Residual Risk        │               │  §9 Summative Evaluation │
└─────────────────────────┘               └──────────────────────────┘
          ↓                                              ↓
  위험관리 파일 (RMF)                        사용적합성 엔지니어링 파일 (UEF)
          └──────────────────┬──────────────────┘
                             ↓
               기술문서 (Technical Documentation)
               → MFDS: 기술문서 / FDA: eSTAR / MDR: Annex II
```

**핵심 연결 포인트**

1. **IEC 62366-1 §6 → ISO 14971 §5**: 사용 오류(Use Error)에서 발생하는 위험 요소(Hazard)를 위험관리 파일에 입력
2. **ISO 14971 §8 → IEC 62366-1 §7**: 위험 통제 수단 중 "IFU·경고문(라벨)" 결정이 사용 명세서에 반영
3. **ISO 13485 §7.3 → 두 표준**: 설계 및 개발 계획에 Risk Management Plan과 Usability Engineering Plan 포함 의무

## 4. 3개 지역 인정·요구 현황 비교

### 4.1 ISO 13485

| 항목 | MFDS (국내) | FDA (미국) | MDR (EU) |
|---|---|---|---|
| **법적 근거** | 의료기기법 §6, KGMP 고시 제2024-88호 | 21 CFR Part 820 (QMSR, 2026-02-02 발효) | MDR Annex IX §2.2 |
| **요구 표준** | KGMP ≈ ISO 13485 구조 동일, 한국어 문서 추가 의무 | QMSR = FDA 자체 규정 (ISO 13485 조화 의도) | ISO 13485:2016 직접 참조 (EN 버전) |
| **심사 기관** | MFDS 또는 위탁 심사기관 (한국의료기기안전정보원) | FDA QSI (신고의료기기는 미심사) | NB 인증서 + 적합성 평가 보고서 |
| **주요 차이** | 전산화 시스템 밸리데이션 추가 고시 | ARR (Abbreviated Release Record) 추가 요건 | PRRC 지정 의무 (MDR Art.15) |
| **인정 경로** | KGMP 인증 (별도 제출 불필요) | 510(k) QSR 준수 선언 | NB 심사 시 Annex IX 섹션으로 포함 |

### 4.2 ISO 14971

| 항목 | MFDS (국내) | FDA (미국) | MDR (EU) |
|---|---|---|---|
| **법적 근거** | 의료기기 안전에 관한 규칙 §18, 기술문서 작성 가이드 | FDA Guidance "Factors to Consider When Making Benefit-Risk Determinations" | MDR Annex I GSPR §3 (안전·성능), §8 (EMC), §14 (방사선) |
| **요구 버전** | ISO 14971:2019 (또는 KS Q ISO 14971) | ISO 14971:2019 권고 (강제 아님, 2007도 수용 가능 — eSTAR 지침서 참조) | EN ISO 14971:2019+A11:2021 (조화표준 — ZA Annex 준수 필수) |
| **제출 서류** | 위험관리 계획서 + 위험관리 보고서 (기술문서 내 포함) | eSTAR §Risks to Health 섹션 (RMF 요약 첨부) | RMF 전체 + GSPR 트레이서빌리티 매트릭스 |
| **EU 특이사항** | — | — | ZA Annex: Presumption of Conformity (조화표준 적용 시 GSPR 만족 추정) |

### 4.3 IEC 62366-1

| 항목 | MFDS (국내) | FDA (미국) | MDR (EU) |
|---|---|---|---|
| **법적 근거** | 의료기기 안전에 관한 규칙 §18; 사용자 안전 확보 요건 | FDA Guidance "Applying Human Factors and Usability Engineering to Medical Devices" (2016) | MDR Annex I GSPR §5 (인체공학), §22 (IFU) |
| **요구 버전** | IEC 62366-1:2015+AMD1:2020 | IEC 62366-1:2015 (AMD1 반영 권고) | EN IEC 62366-1:2015+AC:2016+AMD1:2021 (조화표준) |
| **제출 서류** | UEF 요약 (기술문서 내 포함) | Human Factors Report (총괄적 평가 보고서) | UEF 전체 (Annex II §6 연계) |
| **핵심 시험** | 사용자 시험 보고서 필요 | Summative Evaluation Report (최소 15명 권장) | Summative Evaluation + Hazardous Situations 기록 |

## 7. 3지역 비교 매트릭스 — 표준 인정 방식 요약

| 표준 | MFDS | FDA | MDR |
|---|---|---|---|
| **ISO 13485** | KGMP 적용 (구조 동일, 한국어 의무) | QMSR (21 CFR 820) — 구조 조화, 별도 강제 규정 | EN ISO 13485 직접 참조 (Annex IX §2.2) |
| **ISO 14971** | KS Q ISO 14971 (권고) | ISO 14971:2019 (강제 아님, 권고) | EN ISO 14971:2019+A11:2021 (조화표준, Presumption of Conformity) |
| **IEC 62366-1** | IEC 62366-1:2015+AMD1 (권고) | IEC 62366-1:2015 (권고, FDA Guidance 보완) | EN IEC 62366-1:2015+AC:2016+AMD1:2021 (조화표준) |
| **심사 방식** | 기술문서 내 포함 + GMP 심사 | eSTAR 제출 + QSR 준수 선언 | NB 적합성 평가 (Annex IX/X) + Technical File |
| **조화표준 효과** | 법적 구속력 없음 | 조화표준 아님 (Recognized Consensus Standard) | Presumption of Conformity (추정 조항) |

---

## 8. 자주 나오는 심사 지적사항

### ISO 13485

1. **§7.3 설계 변경 이력 누락**: 설계 변경 시 영향 평가(impact assessment) 기록 없음
2. **§7.4 공급업체 자격 기준 불명확**: Approved Supplier List 갱신 누락
3. **§8.5.2 CAPA 효과성 검증 생략**: 시정 조치 후 재발 방지 확인 기록 없음

### ISO 14971

1. **위험 추정 근거 불투명**: 확률(P1·P2) 수치 산출 근거(문헌·이력 데이터) 미제시
2. **전체 잔류위험 benefit-risk 평가 형식화**: "Acceptable" 기재만 있고 판단 근거 없음
3. **EN ISO 14971:2019+A11:2021 미적용 (EU)**: 2007 버전 사용 시 Presumption of Conformity 불인정

### IEC 62366-1

1. **총괄적 평가 참가자 기준 불충분**: 실사용 대표 사용자 아닌 내부 직원만 참여
2. **Use Error와 Design Flaw 혼동**: 사용 오류를 설계 결함으로 처리하지 않아 RMF 연계 누락
3. **형성적 평가 결과 반영 추적성 없음**: 개선 조치 여부를 UEF에서 추적 불가

## 9. 실습 과제

### 실습 A: ISO 13485 — 설계 변경 영향 평가 (45분)

**시나리오**: Handheld X-ray Source의 배터리 공급업체를 A사 → B사로 변경.

수행 내용:
1. ISO 13485 §7.3.9 설계 변경 기준에 따라 변경 분류 (Minor / Major / Re-design)
2. 영향을 받는 문서·시험 목록 작성 (7.4 공급업체 자격, 62133-2 배터리 안전 시험 등)
3. CAPA와의 연계 여부 판단
4. MFDS 변경 허가 필요 여부 판단 기준 적용

---

### 실습 B: ISO 14971 — FMEA 초안 작성 (45분)

**시나리오**: 촬영실 GUI SW의 kVp 입력 화면 신규 설계.

수행 내용:
1. Hazard Identification: 사용자 오입력 → 방사선 과다 피폭 시나리오 도출
2. 위험 추정: Severity(S) × Probability(P) 매트릭스 작성 (ISO 14971 Annex C 기반)
3. 위험 통제: 입력 범위 제한(안전 설계) → 경고음(방호 수단) → IFU 경고문(정보 제공) 순위 결정
4. 잔류위험 재평가 및 수용 기준 적용

---

### 실습 C: IEC 62366-1 — Use Error 목록 작성 (30분)

**시나리오**: X-ray Detector의 무선(WiFi) 연결 인터페이스 설계 검토.

수행 내용:
1. 의도된 사용자·환경 정의 (SoU 초안)
2. 3개 이상의 Use Error 시나리오 도출
3. 각 시나리오의 Hazardous Situation → Harm 경로 작성
4. ISO 14971 RMF로 이관할 Use-Related Hazard 목록 작성
