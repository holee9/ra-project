## 1. 학습 목표 (Capstone Competencies)

12주 과정 이수 후 다음 역량을 실증적으로 수행할 수 있어야 한다.

1. **규제 경로 판단**: X-ray 3개 제품을 3개 지역(MFDS·FDA·MDR)에서 올바른 등급·경로로 분류하고, 허가/인증/신고·510(k)/De Novo·CE Marking 절차를 선택한다.
2. **기술문서 설계**: 단일 Common Core 기술문서(STED) 구조를 설계하고, 지역별 Annex(MFDS 기술문서·FDA eSTAR·MDR Annex II/III)로 분기시키는 전략을 수립한다.
3. **표준·시험 매핑**: 3개 제품 각각에 적용되는 IEC 60601 시리즈·IEC 62220·IEC 62304·ISO 14971·ISO 13485 요건을 연결하고 시험 계획을 작성한다.
4. **심사 대응 문서 작성**: FDA AI(Additional Information) 회신, MDR NB Deficiency Letter 답변, MFDS 보완자료를 주어진 시나리오 기반으로 작성한다.
5. **QMS·PMS 통합**: ISO 13485 기반 QMS 절차서 체계와 MDR/FDA/MFDS PMS Plan을 제품에 맞게 구성한다.
6. **글로벌 동시 제출 전략**: MDSAP 활용 방안, 3지역 심사 일정 조율, 공통 서류 최적화를 포함한 글로벌 로드맵을 제시한다.

### 시나리오 A: Handheld X-ray Source — 국내 MFDS 허가 전 과정

#### A-0. 배경

> 귀사는 배터리 구동 방식 Handheld X-ray Source (고정전압 60 kVp, 최대 튜브전류 2 mA, 충전식 Li-ion 배터리)를 개발하여 국내 시판 허가를 추진한다. 해외 동등품(FDA 510(k) 허가, Class II) 존재. 국내 미허가 신제품.

#### A-1. 등급 판단 및 허가 경로

| 판단 항목 | 결론 | 근거 |
|---|---|---|
| 의료기기 해당 여부 | 해당 | 의료기기법 §2(1): 진단 목적 방사선 발생 장치 |
| 등급 | **3등급** | 시행규칙 별표 1: 방사선 발생 의료기기 (잠재적 위해 높음) |
| 허가 경로 | **품목 허가** (MFDS 직접 심사) | 3등급 = 반드시 MFDS 직접 허가 |
| KGMP 적용 | 제조 허가 필수 (제조자·수입자 구분) | 의료기기법 §10, §17 |
| 방사선 안전 기준 | 21 CFR 1020.30 대비 국내 **「진단용 방사선 발생장치의 안전관리에 관한 규칙」(보건복지부·질병관리청)** 적용 |

**실습 과제 A-1**: 아래 표를 완성하라.

| 제출 단계 | 담당 기관 | 제출 서류 | 기한 |
|---|---|---|---|
| 의뢰시험 의뢰 | KOLAS 인정 시험기관 | 시험의뢰서 + 시제품 | 허가 신청 전 |
| 허가 신청 | MFDS 의료기기심사부 | ① ~ ⑩ (기재) | — |
| 제조 허가 | MFDS 지방청 | 시설기준 충족 서류 | 허가 신청 전 또는 동시 |
| 품목 기재 | MFDS | 허가증 발급 후 | 허가 후 |

*모범 답안*: 허가 신청 서류 10종 = ① 허가신청서(별지 제1호) ② 기술문서(사용목적·원리·성능·안전성) ③ 임상자료 또는 동등성 자료 ④ 위험관리파일 ⑤ 규격(사양서) ⑥ 시험성적서(KOLAS 인정) ⑦ 제조(수입) 허가증 사본 ⑧ GMP 적합인정서 ⑨ 라벨·IFU ⑩ 방사선 안전 관련 자료.

#### A-2. 기술문서 구성 계획

Handheld X-ray Source 국내 기술문서 5대 섹션 구성:

```
§1. 사용목적 및 적용대상
    - 사용목적: 외래·이동 환경에서 성인 사지·구강 X선 영상 획득

§4. 안전성
    - IEC 60601-1 Ed.3.2 전기적 안전
    - IEC 60601-1-2:2014+A1:2020 EMC (CISPR11/IEC 61000)
    - ISO 14971:2019 위험관리파일
    - 「방사선 발생장치 안전기준 고시」 누설 방사선 기준 충족
§5. 임상자료
    - 동등성 전략: FDA 510(k) K-series 동등품 적용
    - 문헌 평가: PubMed 10건 (DRL 이하 선량, 진단적 적합성)
```

### 시나리오 B: X-ray Detector — FDA 510(k) 제출 전략

#### B-0. 배경

> 귀사의 고체형(Solid-State) CMOS 기반 X-ray Flat Panel Detector (42 × 42 cm, DQE ≥ 55% @ 1 lp/mm, 14-bit ADC)를 미국 시장에 출시한다. Predicate: K201234 (a-Si TFT FPD, 43 × 43 cm, Class II, 21 CFR 892.1750). 동등성 근거 확보가 핵심.

#### B-1. 510(k) 제출 유형 선택

| 유형 | 해당 여부 | 이유 |
|---|---|---|
| Traditional 510(k) | **선택** | 기술적 차이(CMOS vs a-Si) 있음 → Special 불가, Abbreviated ASTER 미해당 |
| Abbreviated 510(k) | 불해당 | FDA recognized consensus standard로 완전 커버되지 않음 |
| Special 510(k) | 불해당 | 자사 기 허가 변경 건 아님 |

#### B-2. Substantial Equivalence (SE) 비교 매트릭스

| 항목 | 자사 제품 | Predicate (K201234) | 동등 여부 | 근거 |
|---|---|---|---|---|
| 사용목적(IFU) | 성인·소아 X선 디지털 영상 획득 (흉부·사지·골반) | 성인·소아 X선 디지털 영상 획득 | **동등** | IFU 직접 비교 |
| 기술 원리 | CMOS Flat Panel Detector | a-Si TFT Flat Panel Detector | **차이** | 검출기 재질·회로 상이 |
| 이미지 크기 | 42 × 42 cm | 43 × 43 cm | **동등 수준** | 임상 사용 유효 촬영 영역 동일 |
| 해상도 | 3.5 lp/mm (143 μm pitch) | 3.1 lp/mm (160 μm pitch) | **동등 이상** | 성능 우위, SE 지지 |
| DQE | 55% @ 1 lp/mm (IEC 62220-1-1) | 50% @ 1 lp/mm | **동등 이상** | 성능 우위, SE 지지 |
| 전기적 안전 | IEC 60601-1 Ed.3.2 | IEC 60601-1 Ed.3.1 | 동등(상위 버전) | Predicate 기준 이상 충족 |
| EMC | IEC 60601-1-2:2014+A1 | IEC 60601-1-2:2007 | 동등(상위 버전) | 상위 기준 충족 |
| **결론** | | | **SE 주장 가능** | 기술적 차이가 새로운 안전·유효성 우려 미제기 |

#### B-3. 기술적 차이 안전성 입증 전략 (CMOS vs a-Si)

CMOS 기반 기술적 우위 근거 (FDA AI 예상 대응):

1. **DQE 비열등성**: CMOS 55% ≥ a-Si 50% → 동등 이상 성능 (IEC 62220-1-1 시험성적서)
2. **방사선 선량 동등성**: 동일 임상 선량에서 동일 수준 진단 영상 품질 달성 → DRL 이하 확인
3. **신뢰성**: MTBF 데이터, 온도/습도 가속 시험 (IEC 60068-2-1/2-2 기반)
4. **사이버보안**: CMOS 직렬 인터페이스(LVDS/USB) → TLS 1.3 암호화, 펌웨어 업데이트 무결성 검증 (FDA 524B 준거)

#### B-4. eSTAR 제출 패키지 체크리스트

```
□ Form FDA 3514 (510(k) Cover Sheet)
□ Truthful and Accurate Statement (Form FDA 3575)
□ Device Description (재질·구성·작동 원리·사용목적)
□ Indications for Use Statement (Form FDA 3881)
□ Substantial Equivalence Comparison Table
□ Performance Testing Summary
   - IEC 62220-1-1 DQE 시험성적서 (NVLAP/A2LA 인정 시험소)
```

### 시나리오 C: 촬영실 GUI SW — MDR 적합성 평가

#### C-0. 배경

> 귀사의 촬영실 GUI SW(DICOM 3.0 기반 영상 수집·처리·디스플레이, PC기반, 독립형 SW)를 EU 시장에 출시한다. 국내·미국 허가는 완료. SaMD(Software as a Medical Device) 해당 여부 및 MDR 등급 판단이 우선.

#### C-1. SaMD 해당 여부 및 MDR 등급 판단

| 판단 단계 | 결론 | 근거 |
|---|---|---|
| 의료기기 해당 여부 | **해당** | MDR Art.2(1): SW가 진단·치료 등 목적에 사용되면 의료기기. 영상 수집·처리는 진단 지원 |
| MDR vs IVDR 구분 | **MDR 적용** | X-ray 영상 수집·표시는 MDR 대상 (체외진단 아님) |
| 등급 판단 (Annex VIII Rule 11) | **Class IIa** | Rule 11: 진단 목적 SW, 중증 악화 결정에 직접 사용되지 않음 (의료인이 최종 판단) |
| 적합성 평가 경로 | **Annex IX** (QMS 기반) | Class IIa → QMS + Technical Documentation |
| EUDAMED 의무 | **의무** | 2026-05-28 전면 시행 (Basic UDI-DI 등록 포함) |

#### C-2. MDR Annex II Technical Documentation 구성

| 섹션 | 주요 내용 | GUI SW 특화 내용 |
|---|---|---|
| §1 Device Description | 제품명·모델·의도 사용목적·사용자·UDI | DICOM 지원 버전, OS 호환성, 하드웨어 요구사항 |
| §2 GSPR Reference | MDR Annex I GSPR 전항 적합성 선언 | §17.2 IT보안, §17.4 사이버보안, §18.8 SW 수명주기 |
| §3 Design and Manufacturing | 설계 이력, 변경 관리, 제조 공정 | Agile/DevSecOps CI/CD 파이프라인, 소스 코드 관리(Git) |
| §4 GSPRs (Annex I) | 각 GSPR에 대한 적합성 방법·표준 | IEC 62304, IEC 62366-1, ISO 14971 맵핑 |
| §5 Benefit-Risk | 임상 편익 vs. 위험 분석 | 오진단 위험 vs. DICOM 표준 영상 처리 효율성 향상 |

#### 실기 과제 2 (20점): 위험 분석 시나리오 대응

**시나리오**: GUI SW가 DICOM 영상 전송 중 패킷 손실로 인해 영상 일부 누락된 채 화면에 표시되는 경우를 ISO 14971:2019 기반으로 위험 분석하라.

| 분석 항목 | 작성 내용 |
|---|---|
| Hazard | (수험자 작성) |
| Hazardous Situation | (수험자 작성) |
| Harm | (수험자 작성) |
| Probability of Occurrence (P1, P2) | (수험자 작성) |
| Severity (S) | (수험자 작성) |
| Risk = P × S | (수험자 작성) |
| Risk Control 조치 | (수험자 작성) |
| Residual Risk | (수험자 작성) |

**모범 답안**:

| 분석 항목 | 내용 |
|---|---|
| Hazard | 네트워크 패킷 손실 |
| Hazardous Situation | 불완전 영상이 정상으로 표시되어 방사선사/의사가 진단에 사용 |
| Harm | 오진단 → 치료 지연 또는 불필요한 추가 촬영(방사선 추가 노출) |
| P1 (위해 발생 가능성) | 3 (드물지 않음, 네트워크 장애 가능) |
| P2 (위해가 Harm으로 이어질 가능성) | 2 (의료인 2차 확인 가능성 있음) |
| S (심각도) | 3 (중등도: 비가역적이나 생명 위협 수준 아님) |
| Risk | P1×P2×S = 18 → 수용 불가 (ALARP 전) |
| 위험 감소 조치 | ① DICOM 전송 완결성 검증 (DIMSE 확인 응답), ② 불완전 영상 자동 경고 표시 (Alert UI), ③ 재전송 자동 요청 로직 |
| Residual Risk | P1×P2=1×1, S=3 → 3 → 수용 가능 (ALARP 달성) |

### 5.3 구술 평가 (30점 · 롤플레이 Q&A)

#### 구술 시나리오 3개 중 추첨 1개

**구술 A: FDA AI Request 대응 (Handheld X-ray Source)**

> *심사관 질문*: "Your device uses a lithium-ion battery. Please provide evidence that the battery safety testing performed is sufficient to demonstrate that the device does not pose an unreasonable risk of fire or explosion under foreseeable conditions of use and misuse."

*기대 답변 핵심 포인트*:
- 시험 표준: IEC 62133-2:2017 (배터리 안전), UL 2054 (선택적 추가)
- 시험 항목: 과충전·과방전·단락·온도 충격·기계적 충격 시험
- 시험 결과: 모든 조건에서 발화·폭발·누액 없음 (성적서 번호 인용)
- 추가 안전 조치: BMS(Battery Management System) 과전류 차단, 온도 모니터링
- 문서 위치: DHF §4.3 Electrical Safety, IEC 62133-2 Test Report [번호]

**구술 B: MDR NB Deficiency — 임상평가 데이터 불충분**

> *NB 심사관*: "The Clinical Evaluation Report submitted does not provide sufficient post-market clinical data for your X-ray Detector. The literature search covers publications up to 2022 only, and there is no PMCF Plan. Please address this deficiency."

*기대 답변 핵심 포인트*:
- 2022년 이후 문헌 검색 업데이트 (PubMed/Embase, PRISMA 프로토콜 적용)
- PMCF Plan 수립 근거: MDCG 2020-7 §4 (IIa 기기 → PMCF 원칙적 필수)
- PMCF 수집 방법: Post-Market Customer Survey + 이상사례 모니터링 + 의료기관 성능 트래킹
- 업데이트 CER 제출 일정: 60일 이내 (NB 합의 기한 준수)

**구술 C: MFDS 보완 요청 — GMP 심사 설계 변경 이력**

> *심사원*: "2025년 1월 PCB 보드 공급업체가 변경되었는데, 설계 변경 관리 절차에 따른 검토·검증 기록이 없습니다. 공급업체 변경이 설계 변경에 해당하는지, 해당 시 필요한 절차는 무엇인지 설명하고 관련 기록을 제시하여 주시기 바랍니다."

*기대 답변 핵심 포인트*:
- 공급업체 변경의 설계 변경 해당 여부: PCB 규격·소재가 동일하면 공급업체 관리(Supplier Control, ISO 13485 §7.4) 대상. 규격 변경 동반 시 설계 변경(§7.3.9) 병행
- 필요 절차: ① 변경 영향 평가(Impact Assessment) → ② 적합성 유지 여부 판단 → ③ 필요 시 재검증 시험(재성능 시험) → ④ DHF 갱신 → ⑤ 규제 당국 보고 여부 판단
- 제출 가능 문서: Supplier Qualification Report (신규 공급업체), PCB 동등성 비교표, 재시험 성적서 (해당 시)
