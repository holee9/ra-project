# MFDS 디지털의료기기 관련 가이드라인 6종 제·개정 요약

> **발행일**: 2025-05-07 (BACKLOG 표기 "2025.01"은 디지털의료제품법 시행일 2025-01-24 기준 표기로 추정; 실제 가이드라인 공표일은 2025-05-07)
> **법적 근거**: 디지털의료제품법 (2024-01-23 제정, 2025-01-24 시행) + 하위 고시 2종

## 1. 배경 및 규제 체계

### 1-A. 디지털의료제품법 시행 경과

| 일자 | 내용 |
|---|---|
| 2024-01-23 | 디지털의료제품법 제정 (법률 제19862호) |
| 2025-01-24 | 디지털의료제품법 전면 시행 |
| 2025-04-07 | 고시 제2025-23호 「디지털의료제품 분류 및 등급 지정 등에 관한 규정」 제정·시행 |
| 2025-04-15 | 고시 제2025-25호 「디지털의료제품 허가·인증·신고·심사 및 평가 등에 관한 규정」 제정·시행 |
| 2025-05-07 | **가이드라인 6종 제·개정 공표** (본 문서 대상) |
| 2026-01-24 | 전자적 표시기재(소프트웨어 화면) 의무화 시행 예정 |

### 1-B. 디지털의료제품 분류 체계

```
디지털의료제품
├── 디지털의료기기
│   ├── 디지털의료기기소프트웨어
│   │   ├── 내장형 (Embedded): 하드웨어에 설치·연결되어 제어·구동·데이터처리
│   │   ├── 독립형 (Standalone): 범용 컴퓨터 등에서 독립 운영
│   │   └── 기타 식약처장 지정
│   └── 그 외 디지털의료기기 (하드웨어 포함 시스템)
├── 디지털융합의약품 (의약품 + 디지털의료기기 조합)
└── 디지털의료·건강지원기기 (비의료기기 웰니스 범주)
```

**판단 기준**: 디지털기술(지능정보·로봇·정보통신 등)이 적용되고, 질병의 진단·치료·예후 관찰·치료반응·부작용 모니터링 목적이면 디지털의료기기에 해당.

## 2. 6종 가이드라인 개요

| 번호 | 구분 | 가이드라인명 | 주요 적용 대상 |
|---|---|---|---|
| 1 | **제정** | 디지털의료기기소프트웨어 허가·심사 가이드라인 | 내장형·독립형 소프트웨어 전반 공통 기본서 |
| 2 | 개정 | 의료기기 소프트웨어 허가·심사 가이드라인 | **내장형** 소프트웨어 + 의료기기 연동 앱·웹 |
| 3 | 개정 | 인공지능기술이 적용된 디지털의료기기 허가·심사 가이드라인 | MLMD(Machine Learning-enabled Medical Device) 전반 |
| 4 | 개정 | 가상융합기술이 적용된 디지털의료기기 허가·심사 가이드라인 | VR·AR·MR 기술 적용 의료기기 |
| 5 | 개정 | 디지털치료기기 허가·심사 가이드라인 | **독립형** SW 중 질병 예방·관리·치료 목적 제품 |
| 6 | 개정 | 인공지능기술이 적용된 디지털의료기기 임상시험방법 설계 가이드라인 | AI 적용 의료기기 임상시험계획 승인 |

> **구조**: 가이드라인 1번(신제정)이 기본서 역할, 2~6번은 기본서를 보완하는 계층 구조.

## 3. 가이드라인별 핵심 내용

### 3-1. 디지털의료기기소프트웨어 허가·심사 가이드라인 (신제정)

**목적**: 내장형·독립형 소프트웨어 공통 적용 기본 허가심사 지침 표준화

**핵심 규율 사항**:
- 소프트웨어 형태적·기능적 특성 분류 기준
- **안정성 등급** (Safety Class) 산정 기준 — IEC 62304 안전등급 연동
- 허가신청서 및 첨부서류 표준 작성방법
- 심사 필요 **변경사항** 목록 (신청서 재제출 트리거)
- 첨부서류 목록: 소프트웨어 적합성 확인보고서, 소프트웨어 검증·유효성확인(V&V) 자료 등

### 3-2. 의료기기 소프트웨어 허가·심사 가이드라인 (개정)

**주요 개정 내용**:
- 적용 범위를 **내장형 소프트웨어**로 한정 (독립형 관련 내용 삭제)
- 의료기기 e분류 품목 관련 내용 삭제 (디지털의료제품법 체계 반영)
- 첨부자료 예시 중 내장형 외 항목 삭제

### 3-3. 인공지능기술 적용 디지털의료기기 허가·심사 가이드라인 (개정)

**주요 개정 내용**:
- 용어·적용범위를 고시 제2025-23호·제2025-25호에 맞게 정비
- 첨단기술 정의 예시 추가 → 디지털의료기기 특성 명확화
- '성능(기능) 또는 특성' 항목에 **디지털기술 세부특성 기재사항** 추가
- 동등비교를 통한 제출자료 범위[별표7], 절차표 삭제 (신고시 체계 반영)

**MLMD 핵심 요건**: 훈련 데이터셋 관리 계획, 알고리즘 유형·구조, 성능평가 방법, 지속학습(Continuously Learning) 여부 신고.

### 3-4. 가상융합기술 적용 디지털의료기기 허가·심사 가이드라인 (개정)

**주요 개정 내용**:
- 용어·적용범위 정비
- **VR·AR·MR 적용 의료기기 해당 여부 판단 흐름도** 추가
- 가상융합기술 적용 소프트웨어 내장 디지털의료기기의 **하드웨어 동등성** 첨부자료 안내
- 임상시험 등 평가 자료 제출 면제 요건 안내

### 3-5. 디지털치료기기 허가·심사 가이드라인 (개정)

**디지털치료기기 해당 여부 3가지 판단 기준**:
1. 독립형 디지털의료기기소프트웨어인지 여부
2. 질병 예방·관리·치료 목적으로 **환자에게 제공**되는지 여부
3. 치료 작용기전의 **과학적 근거**가 있는지 여부

**주요 개정 내용**:
- 판단기준 및 절차표 수정
- '성능(기능) 또는 특성' 예시 추가: 임상적 성능, 주요기능, 기술 구분
- 사용목적·기재방법 예시 확대

### 3-6. AI 적용 디지털의료기기 임상시험방법 설계 가이드라인 (개정)

**주요 개정 내용**:
- 용어·적용범위 정비
- ROC(Receiver Operating Characteristic) Curve 관련 평가변수 내용 삭제 (과도한 제한 완화)

## 5. 자사 3개 제품 영향 맵핑

### 5-1. X-ray Detector (디지털 X선 검출기)

**제품 분류**: 하드웨어 의료기기 + **내장형 디지털의료기기소프트웨어** (검출기 구동·영상처리 SW)

| 적용 가이드라인 | 영향 | 대응 요건 |
|---|---|---|
| 가이드라인 1 (기본서) | ✅ 적용 | SW 안정성 등급 산정 (IEC 62304 Class B 이상 예상), SW 적합성 확인보고서 필수 |
| 가이드라인 2 (내장형) | ✅ 직접 적용 | 허가신청서 내 내장형 소프트웨어 특성 기재, V&V 자료 첨부 |
| 가이드라인 3 (AI) | 조건부 적용 | AI/ML 알고리즘 내장 시 (예: 노이즈 저감, 자동노출제어 ML 적용) → 훈련 데이터셋 관리 계획 추가 요구 |
| 가이드라인 4 (VR/AR) | ❌ 해당 없음 | — |
| 가이드라인 5 (DTx) | ❌ 해당 없음 | 진단 목적으로 환자 직접 제공 아님 |
| 가이드라인 6 (AI 임상) | 조건부 적용 | AI 기반 임상 성능 주장 시 임상시험계획 승인 필요 |

**핵심 액션**:
- 내장 SW 목록 및 기능 분류 → 내장형 해당 여부 확인
- SW 안정성 등급 산정 문서 작성 (SOP에 반영 필요)
- AI 기능 로드맵에 따라 MLMD 해당 여부 사전 판단

### 5-2. Handheld X-ray Source (휴대용 X선 발생기)

**제품 분류**: 하드웨어 의료기기 + **내장형 디지털의료기기소프트웨어** (조사 파라미터 제어 SW)

| 적용 가이드라인 | 영향 | 대응 요건 |
|---|---|---|
| 가이드라인 1 (기본서) | ✅ 적용 | SW 안정성 등급 — 방사선 출력 직접 제어이므로 Class C 가능성 높음 |
| 가이드라인 2 (내장형) | ✅ 직접 적용 | kV/mA/시간 제어 로직 → V&V 필수, SW 변경사항 관리 엄격 적용 |
| 가이드라인 3 (AI) | 조건부 적용 | AEC(Auto Exposure Control) AI화 시 MLMD 요건 추가 |
| 가이드라인 4~6 | ❌ 해당 없음 | — |

**핵심 액션**:
- 방사선 제어 SW 안정성 등급 Class C 판단 근거 문서화
- V&V 계획에 방사선 출력 제어 시나리오 포함
- 배터리 관리 SW 포함 시 IEC 62133-2 안전 요건 연동 확인

### 5-3. 촬영실 GUI SW (방사선 촬영실 워크스테이션 소프트웨어)

**제품 분류**: **독립형 디지털의료기기소프트웨어** (범용 PC에서 독립 운영)

| 적용 가이드라인 | 영향 | 대응 요건 |
|---|---|---|
| 가이드라인 1 (기본서) | ✅ 직접·핵심 적용 | 독립형 SW 기본 허가 신청의 근간 가이드라인 — 신청서 전 항목 준거 |
| 가이드라인 2 (내장형) | ❌ 해당 없음 | 독립형이므로 가이드라인 2 비적용 |
| 가이드라인 3 (AI) | 조건부 적용 | AI 기반 영상 보정·CAD(Computer-Aided Detection) 기능 포함 시 MLMD 요건 |
| 가이드라인 4 (VR/AR) | 조건부 적용 | 증강현실 기반 포지셔닝 가이드 기능 포함 시 VR/AR 가이드라인 추가 적용 |
| 가이드라인 5 (DTx) | ❌ 해당 없음 | 치료 제공 목적 아님, 방사선 기사·의사 대상 장비 |
| 가이드라인 6 (AI 임상) | 조건부 적용 | AI CAD 임상 주장 시 임상시험계획 승인 |

**핵심 액션**:
- 가이드라인 1 기반 독립형 SW 허가신청서 작성 프레임 수립
- SW 적합성 확인보고서 구조 설계 (IEC 62304 Life Cycle 산출물 매핑)
- 전자적 표시기재(화면 내 라벨) 설계 — 2026-01-24 의무화 전 준비 필요

## 6. 사실/해석/행동 정리

### 사실 (Fact)
- 디지털의료제품법은 2025-01-24 시행, 하위 고시 2종(제2025-23호·제2025-25호)은 2025-04-07/15 시행.
- 6종 가이드라인은 2025-05-07 공표. 신제정 1종 + 개정 5종.
- 가이드라인 구조: 기본서(1종) + 제품 유형별 보완서(5종) 계층 구조.
- 독립형 SW는 가이드라인 1 기준, 내장형 SW는 가이드라인 1+2 기준.
- 전자적 표시기재 의무화: 2026-01-24 시행 예정.

### 해석 (Interpretation)
- MFDS는 디지털의료기기 규제를 하드웨어 기반 의료기기법 체계에서 분리하여 디지털의료제품법 독자 체계로 전환. SW-as-a-Medical-Device 개념을 법령 수준에서 명문화.
- 내장형/독립형 이분법은 FDA SaMD 개념과 유사하나, 한국은 별도 법률로 분리한 점이 구별됨.
- AI 가이드라인의 ROC curve 삭제는 단일 지표 요건 완화로 현실적 임상 설계 유연성 확보.

### 행동 지시 (Action)
1. **즉시**: 자사 3개 제품 각각의 내장형/독립형 분류 확정 → 해당 가이드라인 세트 확정.
2. **단기 (3개월)**: 허가신청서 초안 템플릿에 고시 제2025-23호·제2025-25호 및 가이드라인 1(기본서) 반영.
3. **중기 (6개월)**: SW 안정성 등급 산정 SOP 수립, V&V 계획 및 소프트웨어 적합성 확인보고서 초안 작성.
4. **2026-01-24 전**: 촬영실 GUI SW 화면 내 전자적 표시기재(제품명, 버전, 용도, 금기사항 등) 설계 완료.
