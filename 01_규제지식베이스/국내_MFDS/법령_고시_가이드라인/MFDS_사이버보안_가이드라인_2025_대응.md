# MFDS 의료기기 사이버보안 가이드라인 (2025.01 개정) 대응 가이드

## 1. 개정 개요

| 항목 | 내용 |
|---|---|
| 문서명 | 의료기기의 사이버보안 허가·심사 가이드라인 (민원인 안내서) |
| 개정일 | 2025-01-10 |
| 법적 근거 | 「디지털의료제품법」(2025-01-24 시행), 의료기기법 시행규칙 |
| 주무기관 | 식품의약품안전평가원 의료기기심사부 |
| 핵심 변경 | 사이버보안 요구사항 **15개 → 35개** 확장 |
| 조화 목적 | IMDRF N60/N73, IEC 81001-5-1 등 국제표준과 조화 → 해외 진출 원활화 |

### 1.1 2025.01 개정 주요 변경사항

1. **요구사항 세분화**: 기존 15개 요구사항을 35개로 세분·구체화 (IMDRF N60 기반)
2. **적용 대상 명확화**: SW 포함·네트워크 연결 여부 기준으로 사이버보안 적용 의무 명확히 구분
3. **IMDRF 3종 가이드라인 한글화 도입** (2025-04-07 배포):
   - N60:2020 — 의료기기 사이버보안 원칙 및 실무 (기반 문서)
   - N70:2023 — 레거시 의료기기의 사이버보안 원칙 및 실무
   - N73:2023 — 의료기기 사이버보안을 위한 소프트웨어 자재명세서(SBOM) 원칙 및 실무
4. **SBOM 제출 의무화**: 소프트웨어 구성요소 목록 요구 (N73 기준)
5. **전 수명주기(TPLC) 접근**: 설계→제조→판매후(post-market)까지 보안 관리 요구

---

## 2. 사이버보안 적용 대상 판정

MFDS는 다음 조건 중 하나 이상 해당 시 사이버보안 허가 제출 의무 부과:

| 조건 | 판정 기준 |
|---|---|
| SW 포함 | 제품 내 펌웨어·소프트웨어 존재 시 |
| 네트워크 연결 | 유선(USB·LAN·DICOM)·무선(Wi-Fi·Bluetooth) 연결 가능 시 |
| 원격 접근 | 원격 진단·업데이트·제어 기능 존재 시 |

> **참고**: 디지털의료제품법상 "디지털의료기기"에 해당하면 사이버보안 요구사항 자동 적용.

## 3. 허가 제출 서류 5종 (2025.01 기준)

| # | 서류명 | 내용 |
|---|---|---|
| 1 | **사이버보안 요구사항 체크리스트** | RA-01~RA-35 (35개 항목) 충족 여부 항목별 점검 |
| 2 | **사이버보안 위험관리 보고서** | 위협 모델 + 사이버보안 위험평가 + SBOM + 취약성 평가 + 미해결 이상 평가 + 추적성 |
| 3 | **시험 성적서 (Test Report)** | 보안 통제 구현 증빙 (침투시험·취약점 스캐닝 등) |
| 4 | **라벨링 (Labeling)** | 사용자 보안 안내문·업데이트 정책·IFU 사이버보안 섹션 |
| 5 | **사이버보안 관리 계획** | 업데이트·패치·CVD(Coordinated Vulnerability Disclosure) 절차 |

### 3.1 사이버보안 위험관리 보고서 구성

```
위험관리보고서 (Cybersecurity Risk Management Report)
├── 위협 모델 (Threat Model)
│   ├── 자산 식별 (Assets)
│   ├── 위협 열거 (STRIDE 등)
│   └── 공격 경로 분석
├── 사이버보안 위험평가
│   ├── 위험 등급 산정 (심각도 × 가능성)
│   └── 잔여 위험 수용 근거
├── SBOM (소프트웨어 자재명세서)
│   ├── 자체 개발 컴포넌트
│   ├── 제3자 상용 SW
│   └── 오픈소스 컴포넌트 (라이선스 포함)
├── 취약성 평가 (Vulnerability Assessment)
│   ├── CVE/NVD 취약점 스캔 결과
│   └── 완화 조치
├── 미해결 이상 평가 (Unresolved Anomaly Assessment)
│   └── 허용 이상 목록 + 위험 수용 근거
└── 추적성 (Traceability)
    └── 보안 요구사항 ↔ 설계 ↔ 시험 결과 매핑
```

### 3.2 RA 요구사항 35개 주요 카테고리

2025.01 개정으로 IMDRF N60 기준 35개로 확장된 요구사항은 아래 8개 카테고리로 분류 (정확한 RA-01~RA-35 번호는 공식 가이드라인 PDF 참조):

| 카테고리 | 주요 요구사항 |
|---|---|
| **보안 설계 원칙** | Secure by design, Defense-in-depth, Least privilege |
| **인증·접근 제어** | 사용자 인증, 역할 기반 접근, 세션 관리 |
| **암호화** | 저장 데이터·전송 데이터 암호화, 키 관리 |
| **소프트웨어 무결성** | 코드 서명, 부트 무결성 검증, SBOM 관리 |
| **네트워크 보안** | 포트 최소화, 방화벽, TLS 버전 요건 |
| **취약점 관리** | CVE 모니터링, 패치 주기, CVD 정책 |
| **감사·로그** | 보안 이벤트 로깅, 로그 보호, 접근 이력 |
| **업데이트·유지보수** | 보안 업데이트 메커니즘, 원격 관리 보안 |

## 4. IMDRF 3종 가이드라인 요약

### 4.1 N60:2020 — 의료기기 사이버보안 원칙 및 실무

- **TPLC(Total Product Life Cycle) 프레임워크**: 설계→개발→제조→판매→판매후→폐기 전 단계 보안 관리
- **8대 보안 원칙**: 보안 설계(Security by Design), 위험관리 통합, SBOM, 취약점 공개 정책(CVD), 제품 보안 사고 대응(PSIRT), 보안 업데이트, 교육·훈련, 이해관계자 협력
- **MDM(Medical Device Manufacturer) 책임**: 전 수명주기 사이버보안 계획 수립·유지

### 4.2 N70:2023 — 레거시 의료기기 사이버보안

- **레거시 기기 정의**: 제조사 지원 종료 또는 현재 사이버보안 위협으로부터 합리적 보호 불가 기기
- **의무**: 레거시 여부 판정 기준 문서화, 현장 완화 조치(compensating controls), 단계적 교체 계획
- **자사 적용**: 단종 예정 구형 X-ray 시스템 연계 시 N70 준수 필요

### 4.3 N73:2023 — SBOM 원칙 및 실무

- **SBOM 최소 구성요소**: 컴포넌트명·버전·공급자·고유 식별자(PURL/CPE)·라이선스·관계(dependency)
- **포맷**: SPDX 2.x 또는 CycloneDX 1.4+ 권장 (MFDS 특정 포맷 미지정, FDA와 동일)
- **HCP(의료기관) 공유**: SBOM을 의료기관 요청 시 제공할 수 있는 메커니즘 구축 필요

## 6. 자사 3개 제품 영향 맵핑

### 6.1 X-ray Detector

| 항목 | 내용 |
|---|---|
| **사이버보안 적용 여부** | ✅ 해당 (SW 포함 + DICOM/네트워크 연결) |
| **IEC 62304 Safety Class** | Class B |
| **주요 연결 인터페이스** | DICOM (TCP/IP), USB, RIS/HIS 연동 |
| **고위험 영역** | DICOM 프로토콜 취약점, 환자 데이터 도용 |
| **SBOM 대상** | Detector 펌웨어 + DICOM SW 라이브러리 |
| **MFDS 제출 필요 서류** | 5종 전체 + RA-01~35 체크리스트 |
| **특이사항** | Detector 자체는 네트워크 종단점이므로 Segmentation 설계 권고 |

### 6.2 Handheld X-ray Source

| 항목 | 내용 |
|---|---|
| **사이버보안 적용 여부** | ⚠️ 조건부 (펌웨어 포함 여부·연결 기능에 따라 판정) |
| **IEC 62304 Safety Class** | Class C (방사선 제어 SW) |
| **주요 연결 인터페이스** | USB(펌웨어 업데이트)·Bluetooth(옵션) |
| **고위험 영역** | 방사선 출력 제어 파라미터 변조, 무단 원격 활성화 |
| **SBOM 대상** | 방사선 제어 펌웨어·업데이트 모듈 |
| **MFDS 제출 필요 서류** | 연결 기능 있으면 5종 전체; 연결 없으면 축소 가능 (MFDS 사전 확인 권고) |
| **특이사항** | Bluetooth·Wi-Fi 미탑재 단순 유선 기기는 적용 범위 완화 가능 |

### 6.3 촬영실 GUI SW

| 항목 | 내용 |
|---|---|
| **사이버보안 적용 여부** | ✅ 해당 (IEC 62304 Class C + 병원 네트워크 연결 필수) |
| **IEC 62304 Safety Class** | Class C |
| **주요 연결 인터페이스** | LAN/Wi-Fi, DICOM, RIS/HIS, PACS |
| **고위험 영역** | 환자 PHI 노출, 랜섬웨어 경로, 방사선 프로토콜 변조 |
| **SBOM 대상** | GUI SW 전체 + OS 컴포넌트 + 3rd party 라이브러리 |
| **MFDS 제출 필요 서류** | 5종 전체 (가장 높은 수준 요구) |
| **특이사항** | SaMD(단독 디지털의료기기SW) 분류 시 디지털의료제품법 추가 요건 적용 |

## 7. 제출 준비 체크리스트 (MFDS 2025.01 기준)

### 사전 준비 (설계 단계)
- [ ] 제품별 사이버보안 적용 여부 판정 (SW 포함·네트워크 연결 확인)
- [ ] 보안 요구사항 명세 (SRS에 RA-01~35 매핑)
- [ ] 위협 모델링 수행 (STRIDE 또는 TARA 방법론)
- [ ] SBOM 초안 작성 (N73 기준; SPDX 또는 CycloneDX 포맷)
- [ ] CVD(취약점 공개 정책) 수립

### 시험 단계
- [ ] 취약점 스캐닝 (자동화 도구)
- [ ] 침투 시험 (수동 검증)
- [ ] 보안 통제 구현 확인 (암호화·인증·로깅 검증)
- [ ] 시험 성적서 작성

### 허가 제출 단계
- [ ] RA-01~35 체크리스트 완성 (미충족 항목 근거 기술)
- [ ] 사이버보안 위험관리 보고서 완성
- [ ] 라벨링 검토 (IFU 사이버보안 섹션 포함 여부)
- [ ] 사이버보안 관리 계획 확정

### 판매후 (Post-market)
- [ ] CVE/NVD 정기 모니터링 체계 운영
- [ ] 보안 패치 배포 프로세스 가동
- [ ] 이상 사례 보고 연계 (CVD → MFDS 보고 기준 확인)
