# MFDS 의료기기 이상사례·부작용 보고 절차

## 1. 법적 근거

| 법령 | 조항 | 내용 |
|---|---|---|
| 의료기기법 | 제31조 제1항 | 부작용관리: 제조·수입업자 보고 의무 |
| 의료기기법 시행규칙 | 제51조 | 부작용 보고 기한·방법·양식 |
| 의료기기 이상사례 보고 가이드라인 | 2024-05-27 개정판 | 실무 세부 절차 |

> **디지털의료제품법(2025-01-24 시행, 식약처 소관 — 법·시행령·시행규칙, 국가법령정보 확인)**: GUI SW가 *디지털의료기기*(지능정보기술 등 적용 의료기기)에 해당하면 동법의 안전관리·부작용(안전성정보) 규정이 의료기기법과 병행 적용된다. GUI SW의 디지털의료기기 해당 여부는 제품의 지능정보기술 적용 여부로 분류 판단(제품 분류 확인 필요).

---

## 2. 핵심 용어 정의

| 용어 | 정의 |
|---|---|
| **이상사례** (Adverse Event) | 의료기기 사용과 관련하여 발생한 의도하지 않은 모든 사건 (제품과의 인과관계 불문) |
| **부작용** (Side Effect) | 정상적인 사용방법에 따라 사용할 때 발생한 의도되지 아니한 결과 |
| **중대한 이상사례** (Serious Adverse Event) | 아래 4가지 중 하나 해당: ① 사망, ② 생명 위협, ③ 입원 또는 입원 기간 연장, ④ 지속적·중대한 불구·기능저하 |
| **예측 불가능한 이상사례** | 허가된 IFU 또는 요약 정보에 기재되지 않은 이상사례 |

## 3. 보고 주체 및 대상

### 3.1 보고 주체
- **제조업체** (국내 제조): 1차 보고 의무자
- **수입업체**: 외국 제조업체 대신 국내 보고 의무자
- **의료기관**: 중대한 이상사례 발생 시 자발적 보고 가능 (의무 보고 아님)
- **소비자/환자**: 자발적 신고 가능

### 3.2 보고 대상 이상사례
| 구분 | 보고 여부 |
|---|---|
| 사망 또는 생명 위협을 초래한 이상사례 | **의무 보고** |
| 입원·입원 연장을 초래한 이상사례 | **의무 보고** |
| 영구적 기능 저하·불구를 초래한 이상사례 | **의무 보고** |
| 비중대 이상사례 (재발 위험 없음) | 기록 유지 (자발 보고 권장) |
| 예측 가능한 부작용 | 기록 유지 |

---

## 4. 보고 기한 (시행규칙 제51조)

| 이상사례 유형 | 보고 기한 |
|---|---|
| **사망** 또는 **생명 위협** | 인지 후 **15일 이내** |
| **중대하고 예측 불가능한** 이상사례 | 인지 후 **15일 이내** |
| **중대하나 예측 가능한** 이상사례 | 인지 후 **90일 이내** |
| **비중대 이상사례** | 인지 후 **90일 이내** (자발적) |

> **주의**: 추가 정보가 있을 경우 추적 보고(Follow-up Report) 제출 필요

## 5. 보고 절차 및 방법

### 5.1 보고 흐름
```
[이상사례 접수/인지]
  - 고객 민원, 현장 피드백, 자체 모니터링
        ↓
[초기 평가] ← 48시간 이내
  - 중대성 판단 (4가지 기준)
  - 예측 가능성 판단 (IFU 기재 여부)
  - 보고 기한 결정
        ↓
[보고서 작성]
  - 의료기기 이상사례 보고서 (식약처 소정 양식)
  - 이상사례 표준코드 적용 (2024-05-27 개정 기준)
        ↓
[보고 제출]
  - 방법: 식품의약품안전처 의약품안전나라 시스템 또는 우편·팩스
  - 수신처: 식품의약품안전처장 (의료기기안전국)
        ↓
[추적 보고] (필요 시)
  - 추가 정보 확인 후 갱신 보고
  - Root Cause 파악 시 원인 분석 결과 포함
        ↓
[내부 CAPA 연계]
  - 이상사례 원인이 제품 결함 → FSCA/회수 검토
  - 재발 방지 CAPA 개시
```

### 5.2 보고 방법
| 방법 | 비고 |
|---|---|
| **의약품안전나라** 온라인 시스템 | 권장 방법 (emedi.mfds.go.kr) |
| 팩스 | 긴급 시 병행 가능 |
| 우편 | 온라인 제출 불가 시 |

### 이상사례 표준코드 체계 (2024-05-27 개정)
- ISO/ANSI UMDNS 기반 분류 체계 적용
- 주요 카테고리: 기기 문제(Device Problem), 환자 문제(Patient Problem), 조작자 오류(Operator Error)
- 개정 표준코드 사례집 참조 (식약처 배포)

## 7. 3개 지역 비교

| 항목 | MFDS (한국) | FDA (미국) | MDR (EU) |
|---|---|---|---|
| **보고 근거** | 의료기기법 제31조, 시행규칙 제51조 | 21 CFR 803 | MDR Art. 87 |
| **사망/생명위협 보고 기한** | 15일 | 30일 (MDR30) | 15일 (즉시 위협 시 2일) |
| **중대 예측가능 보고 기한** | 90일 | 30일 | 30일 |
| **보고 수신처** | 식약처장 | FDA/CDRH | EUDAMED + 관할 CA |
| **보고 양식** | 의료기기 이상사례 보고서 | MedWatch 3500A (FDA) | MIR Form 7.3.1 |
| **보고 시스템** | 의약품안전나라 | eMDR / eSubmitter | EUDAMED |
| **회수 연계** | 의료기기법 제32조 | 21 CFR 806 | MDR Art. 83 |

## 8. 자사 3개 제품 적용

| 제품 | 주요 이상사례 유형 | 주의사항 |
|---|---|---|
| **X-ray Detector** | 이미지 아티팩트로 인한 오진, 기계적 파손에 의한 부상 | DQE 허용 범위 이탈 → 즉시 보고 검토 |
| **Handheld X-ray Source** | 방사선 과다 피폭 (선량 초과), 과열·배터리 발화 | 이상사례는 MFDS 보고(의료기기법 §31). 의료용 진단 방사선 안전관리는 **「진단용 방사선 발생장치의 안전관리에 관한 규칙」(보건복지부·질병관리청 소관)** 소관이며 **원자력안전위(NSSC) 아님** → NSSC 병행 보고 일반적으로 해당 없음 |
| **촬영실 GUI SW** | 이미지 처리 오류로 진단 오류, 선량 계산 오류 | SaMD 분류 → 디지털의료기기 이상사례 규정 추가 적용 가능성 |

## 10. 주요 실수 방지 체크리스트

- [ ] 이상사례 인지일 vs 발생일 구분하여 정확히 기재 (보고 기한 기산점은 **인지일**)
- [ ] 이상사례 표준코드 최신 버전 (2024-05-27 개정) 적용 확인
- [ ] 중대성 4가지 기준 모두 검토 (사망/생명위협/입원/불구)
- [ ] IFU 기재 여부 확인하여 예측 가능성 판단
- [ ] 의료용 진단 방사선 기기: 안전관리는 보건복지부·질병관리청(「진단용 방사선 발생장치의 안전관리에 관한 규칙」) 소관 — 원자력안전위(NSSC) 병행 보고 일반적으로 해당 없음(이상사례는 MFDS 단일 보고)
- [ ] SaMD인 경우 디지털의료기기 관련 추가 규정 확인 (GUI SW 해당)
- [ ] 추적 보고가 필요한 경우 기한 내 제출 여부 추적 관리
- [ ] 이상사례 → FSCA 전환 필요 여부 별도 판단
- [ ] 모든 이상사례 기록 보존 (비보고 대상 포함)
