# 의료기기 사이버보안 통합 가이드 (X-ray System)

> 자사 제품 중 SW 포함 / 네트워크 연결 / 무선 기능 있는 모든 제품에 적용.
> 웹 리서치 교차검증 반영 – 2026-04-22.

## 1. 지역별 규제 요약

### 1.1 미국 FDA (가장 강력)
- **법적 근거**: FD&C Act **Section 524B** (2022 Omnibus Consolidated Appropriations Act 추가)
- **"Cyber Device" 정의**:
  1. SW 포함 또는 SW가 기기 기능에 validate된 경우
  2. 인터넷·네트워크 연결 가능
  3. 사이버보안 위협에 노출 가능
- **제출 의무 (법적)**:
  - SBOM (**Software Bill of Materials**) – 자체·3자·오픈소스 전부
  - Threat Model (전체 threat modeling exercise 수행 증빙)
  - Cybersecurity Risk Assessment
  - Cybersecurity Management Plan
  - Vulnerability Management Plan
- **Final Guidance**: "Cybersecurity in Medical Devices: Quality System Considerations and Content of Premarket Submissions" (**2023-09-27**)

### 1.2 유럽 EU MDR
- **GSPR** Annex I 관련 조항 (§14, §17, §18)
- **MDCG 2019-16 Rev.1**: Cybersecurity 가이던스
- **IEC 81001-5-1:2021** 조화표준화 진행 (사용 권장)
- **NIS 2 Directive** 간접 영향 (연결 시)

### 1.3 국내 MFDS
- **「의료기기 사이버보안 허가·심사 가이드라인」 개정판 (2025.01)**
- 「디지털의료제품법」(2025-01-24) 하위 규정 연계
- 제출 자료: Threat Model, Risk Analysis, SBOM, 검증 결과, 업데이트 정책

## 3. SBOM (Software Bill of Materials)

### 3.1 정의
제품 내 모든 SW 컴포넌트와 버전·라이선스·공급원 목록.

### 3.2 FDA 요구 형식 권장
- **SPDX** (Linux Foundation)
- **CycloneDX** (OWASP)
- **SWID Tags** (ISO/IEC 19770-2)

### 3.3 포함 범위 (FDA 2023)
- Manufacturer-developed components
- Third-party commercial/licensed SW
- Open-source SW
- Custom SW
- 의존성 트리 (직접 + 전이적 의존성)

### 3.4 메타데이터 필수 항목
- Component name, version
- Supplier, author
- Unique identifier (CPE, PURL 등)
- Relationship (dependency)
- Hash / signature (가능 시)

## 4. Threat Model 구성 (FDA 2023 요구)

### 4.1 권장 방법론
- **STRIDE** (Spoofing, Tampering, Repudiation, Info Disclosure, DoS, Elevation of Privilege)
- **DREAD** (Risk scoring)
- **Attack Tree**
- **Data Flow Diagram (DFD)** 기반

### 4.2 필수 산출물
- 시스템 컨텍스트 다이어그램
- 자산·데이터 식별
- 신뢰 경계 (Trust Boundary)
- 위협 식별·분류
- 대응책(Mitigation) 매핑
- 잔여 위험(Residual Risk) 평가

## 5. Secure Development Lifecycle (SDL) – IEC 81001-5-1 기반

| 단계 | 핵심 활동 |
|---|---|
| 요구사항 | Security requirements 수립, Threat Model 1차 |
| 설계 | Secure design, 신뢰 경계 설정, 암호 설계 |
| 구현 | Secure coding, 코드 리뷰, SAST |
| 검증 | Pen Test, Fuzz Test, Vulnerability Scan, DAST |
| 릴리즈 | SBOM 생성, 릴리즈 승인, 취약점 공개 절차 |
| 유지관리 | 취약점 모니터링, 패치 관리, 고객 공지 |

## 6. Post-Market 의무

### 6.1 FDA
- 알려진 취약점 지속 모니터링 (CVE, ICS-CERT 등)
- 패치 배포 계획
- 공중위생 영향 취약점(CVSS 고위) 시 **사용자 공지** + FDA 통보 가능
- Medical Device Coordinating Council (H-ISAC) 참여 권장

### 6.2 EU
- PMS에 사이버보안 이슈 포함
- Vigilance: 보안 사고로 인한 심각한 위해 → Serious Incident 보고

### 6.3 국내
- 사이버보안 관련 리콜·시정조치 절차 포함
