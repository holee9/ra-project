# FDA RTA (Refuse to Accept) 회피 체크리스트

> **적용 제출 유형**: Traditional 510(k) / Abbreviated 510(k) / Special 510(k) / De Novo  
> **RTA 심사 기간**: 제출 수령 후 **15 calendar days** 이내  
> **근거 규정**: 21 CFR 807.87, FD&C Act Section 513(i)

---

## 1. RTA 개요

### 1.1 RTA란

FDA가 510(k) 제출물이 실질적 동등성(Substantial Equivalence) 검토를 시작하기 위한 **최소 행정적 완전성(minimum threshold of acceptability)**을 충족하는지 평가하는 초기 심사.

- **목적**: 부족한 제출물 조기 발견 → 전체 심사 기간 단축
- **결과**: Accept(수락) 또는 RTA(반려)
- **RTA 통보**: 반려 항목 명시한 체크리스트 동봉

### 1.2 핵심 원칙

| 원칙 | 내용 |
|------|------|
| **존재 여부만 확인** | RTA 심사는 정보의 적절성(adequacy)이 아닌 **존재 여부(presence)**만 평가 |
| **N/A 설명 허용** | 적용 불가 항목은 N/A 사유 제공 시 "충족"으로 간주 |
| **대안 접근 허용** | 권고사항 미준수 시 대안 접근법 + 근거 제공으로 수락 가능 |
| **RTA 응답 기한** | RTA 통보 후 **180일** 내 응답 없으면 510(k) 폐기 간주 |

| 1 | Predicate 기기 번호 미제공 또는 불일치 | 제출 전 CDRH 제품 분류 DB 재확인, 제출물 전체 predicate 일관성 검토 |

| 4 | Indications for Use Form 누락 또는 불일치 | FDA 권장 양식 사용, 라벨링과 동일 문구 확인 |
| 5 | SW 있으나 Software Documentation 미제출 | SW 존재 여부 명시 + FDA SW 가이던스 준수 문서 포함 |
| 6 | 사이버보안 연결성 미명시 | 모든 인터페이스(USB, BT, Ethernet 등) 열거 |

| 9 | 선행 제출 이력 미기재 (Form 3514 Section F 공란) | "선행 제출 없음" 진술 명기 |
| 10 | 영문 아닌 문서 번역본 미첨부 | 외국어 시험 보고서 전체 영문 번역 |

## 5. 자사 3개 제품별 RTA 주의사항

### 5.1 X-ray Detector (Digital Flat Panel Detector)

- **SW 문서**: 이미지 보정 알고리즘(gain correction, defect map), 영상 재구성 SW → SW Documentation Level Enhanced 이상 예상
- **Cybersecurity**: DICOM/HL7 네트워크 연결 있으면 필수 대응 (SBOM, Threat Model)
- **성능 시험**: IEC 62220-1-1 (DQE), IEC 60601-1, IEC 60601-1-2 (EMC) full report 필수
- **Predicate**: 동일 FPD 기술의 기제출 510(k) 확인 (accessdata.fda.gov에서 검색)

### 5.2 Handheld X-ray Source

- **방사선 방출**: 21 CFR 1020.30/1020.40 요건 — RTA 전에 CDRH가 방사선 방출 기기 여부 확인
- **배터리 안전**: 내장 배터리 → IEC 62133-2 데이터 필요, 배터리 명시 필수
- **SW 문서**: 노출 제어 SW, 조사 시간 제어 → Safety-Critical SW로 분류 가능
- **Intended Use**: 진단 방사선 촬영인지 명확히 (치과용·일반용·특수용 구분)

### 5.3 촬영실 GUI SW (SaMD)

- **Product Code 확인**: SW-only 기기는 별도 Product Code 체계 (QMDB, LMF 등)
- **SW 분류**: IEC 62304 Safety Class B 또는 C → Enhanced Level 문서 요구
- **Cybersecurity**: 병원 네트워크 연결 필수적 → Substantial Cybersecurity 문서 필요
- **전기 안전/EMC**: SW-only이면 N/A 가능하나, HW와 통합 시 포함 필요

## 7. PMA Refuse to File (RTF) 참고

PMA의 경우 RTA 대신 **Refuse to File (RTF)** 정책 적용 (21 CFR 814.42(e)).  
X-ray 제품이 PMA 대상(Class III)인 경우 별도 대응 필요. 통상 X-ray Detector, Handheld Source는 Class II이나, 새로운 indication 또는 기술이면 De Novo/PMA 검토 필요.
