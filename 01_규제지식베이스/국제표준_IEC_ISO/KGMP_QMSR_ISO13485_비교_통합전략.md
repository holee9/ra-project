# KGMP vs QMSR vs ISO 13485 — 차이 분석 및 3개 지역 동시 적합 통합 전략

---

## 1. 3개 QMS 요건 체계 개요

| 항목 | KGMP (한국) | QMSR (미국) | ISO 13485 기반 MDR (EU) |
|---|---|---|---|
| 근거 | 의료기기법 제23조, 식약처 고시 제2024-88호 | 21 CFR Part 820 (QMSR Final Rule, 2024-02-02 공포) | MDR (EU) 2017/745 Annex IX Chapter I |
| 발효일 | 2024-12-27 (최신 개정) | **2026-02-02 발효 확정 · 현재 전면 시행 중** | 적용 중 (MDR 2021-05-26 완전 시행) |
| 핵심 표준 | ISO 13485:2016 기반 구조, 한국어 절차서 | ISO 13485:2016 incorporation by reference | EN ISO 13485:2016+A11:2021 조화 (ZA/ZB/ZC) |
| 감독기관 | 식품의약품안전처 (MFDS) + 지방청 현장 실사 | FDA (ORA) — 신규 Inspection Program 7382.850 | Notified Body (NB) + 회원국 당국 |
| 실사 방식 | MFDS 직접 현장심사 (서류 정합성 강조) | 신규 MDMI Guide (내부감사·경영검토 실사 가능) | NB QMS 심사 (Annex IX) + 불시 감사 |
| 인증 형태 | GMP 적합인정서 | FDA QMSR 준수 (등록·목록 제출, FFDCA 기반) | NB 발행 QMS 인증서 (+ Technical Documentation) |
| MDSAP 연계 | MFDS MDSAP 상호인정 (`[검증 필요]` 최신 현황 확인 권고) | FDA MDSAP 수용 (실사 대체 가능) | MDR Annex IX §3.2 — MDSAP 부분 수용 |

## 3. 지역별 추가(고유) 요건 — ISO 13485 초과 사항

### 3-A. QMSR 고유 요건 (FDA)

| 추가 요건 | 근거 조항 | 내용 |
|---|---|---|
| **ARR (Applicable Regulatory Requirements)** | §820.10(b) | ISO 13485 준수만으로 불충분; UDI(21 CFR 830), MDR reporting(21 CFR 803), Corrections & Removals(21 CFR 806) 별도 준수 의무. QMS 절차서에 ARR 준수 활동 통합 필수. |
| **불만 파일 강화** | §820.20(c) | 불만 접수 시 MDR reportable 여부 판단 기록 필수. ISO 13485 §8.2.1보다 상세. |
| **서비스 기록** | §820.35 | 서비스 수행 기록·서비스 보고서 요건 명시. |
| **내부감사 실사** | §820 전반 | 구 §820.180(c) 내부감사·경영검토 비공개 예외 삭제. FDA 실사관이 내부감사 보고서 열람 가능. |
| **계약 멸균·재라벨러 포함** | §820.10(a) | QMSR 적용 대상: 계약 멸균업체, 재라벨러, 외국 초기 유통업체 포함(기존 QSR보다 확대). |
| **용어 정의 우선순위** | §820.3 | ISO 13485 정의 불채택. FD&C Act §201 용어 우선. (예: "manufacturer"→FD&C Act 정의) |
| **감사 프로그램** | 7382.850 | 2026-02-02부터 신규 MDMI (Medical Device Manufacturers Inspection) Guide 적용. QSIT 폐지. |

### 3-B. KGMP 고유 요건 (MFDS)

| 추가 요건 | 근거 | 내용 |
|---|---|---|
| **한국어 문서화** | 의료기기법 §23, 고시 제2024-88호 | 절차서·기록은 한국어 작성 원칙. ISO 표준 언어와 용어 정합성 MFDS 심사 시 엄격 확인. |
| **전산화 시스템 밸리데이션** | 고시 부칙 — ISO 13485 §4.1.6 강화 | QMS에 사용되는 전산화 시스템(ERP, LIMS 등) 밸리데이션 증거 기록 요구. |
| **GMP 적합인정 절차** | 의료기기법 §23, 시행규칙 §47~51 | 제조소 단위 GMP 심사 → 적합인정서 발급. 갱신 주기: 3년 (일반), 혁신의료기기 별도. |
| **현장 실사 서류 정합** | MFDS 심사기준 | 절차서 문장이 ISO 13485 조항 문구와 불일치 시 지적. 유럽 NB 심사보다 형식적 정합 엄격. |
| **디지털의료제품법 연계** | 디지털의료제품법(2025-01-24 시행) | SW 의료기기 QMS 관련 조항 고시 제2024-88호에서 삭제 → 디지털의료제품법령으로 이관. |
| **MFDS 이상사례 보고** | 의료기기법 §31 | 중대한 부작용 발생 시 15일 이내 MFDS 보고. QMS 피드백 절차에 통합 필요. |

### 3-C. MDR (EU) 고유 요건 (Annex IX QMS)

| 추가 요건 | 근거 | 내용 |
|---|---|---|
| **QMS 범위 선언** | MDR Annex IX §3.1 | QMS 적용 범위 명시 및 NB 검토. |
| **PMS Plan·PSUR·PMSR** | MDR Art. 83~86, Annex III | 제품별 PMS Plan 필수. Class IIa+ → PSUR (2~3년 주기), Class I → PMSR. ISO 13485 §8.2.1 PMS와 연동 필수. |
| **Vigilance (Serious Incident)** | MDR Art. 87~92 | 심각한 사고(Serious Incident): 15일 이내 보고. FSCA/FSN: 즉시 통보. QMS 절차서에 통합. |
| **PRRC (Person Responsible for Regulatory Compliance)** | MDR Art. 15 | QMS 내 PRRC 역할·권한 명시. |
| **Eudamed 등록** | MDR Art. 27~34 | UDI 등록, 제품 등록, PMS 데이터 Eudamed 입력. QMS 프로세스에 통합. |
| **NB 불시 감사** | MDR Art. 44(10) | NB 사전 예고 없이 QMS 감사 수행 가능. 내부감사 결과 NB 제출 가능 형태 유지 필요. |
| **Annex IX Chapter II** | MDR Annex IX §4 | Technical Documentation 검토 (Class III + 일부 IIb): QMS와 별도 Tech File NB 검토. |

## 4. 3개 지역 동시 적합 통합 전략

### 4-1. 전략 개요

```
┌─────────────────────────────────────────────┐
│         ISO 13485:2016 코어 QMS              │
│  (KGMP·QMSR·MDR 공통 기반, 37개 절차서)     │
└──────────────┬──────────────────────────────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
 MFDS 레이어  FDA 레이어  EU 레이어
 (한국어 문서  (ARR 준수   (PMS/PSUR
  + GMP심사)   + 불만파일  + Vigilance
               강화)       + PRRC)
```

**핵심 원칙**: ISO 13485 절차서를 "단일 마스터"로 구성하고, 지역별 추가 요건은 별도 Appendix 또는 조항 확장으로 추가. 절차서 중복 작성 금지.

### 4-2. 통합 QMS 구현 단계별 계획

| 단계 | 활동 | 산출물 | 우선순위 |
|---|---|---|---|
| **1단계** | ISO 13485 절차서 37개 마스터 완성 | SOP 세트 (#31 기준) | 즉시 |
| **2단계** | QMSR ARR 준수 Annex 추가 | §820.10(b) ARR 준수 체크리스트 | 1개월 이내 |
| **3단계** | KGMP 한국어 번역·언어 정합 검토 | 한국어 SOP 세트 | 1개월 이내 |
| **4단계** | MDR Vigilance·PMS 절차 통합 | PMS Plan + Vigilance SOP | 2개월 이내 |
| **5단계** | MDSAP 활용 여부 검토 | MDSAP Gap Analysis | 3개월 이내 |
| **6단계** | 통합 내부감사 체계 구축 | 내부감사 계획 (KGMP·FDA·NB 동시 대응) | 3개월 이내 |

### 4-3. MDSAP (Medical Device Single Audit Program) 활용

- **참여국**: 미국(FDA), 캐나다, 브라질, 일본, 호주 (2026년 현재)
- **한국 현황**: MFDS MDSAP 상호인정 협약 체결 여부 `[검증 필요]` (2026년 최신 현황 mfds.go.kr 확인 권장)
- **EU**: MDR Annex IX은 MDSAP를 직접 대체로 인정하지 않음. 단, NB에 따라 MDSAP 결과 참조 가능.
- **전략**: FDA·MFDS 동시 대응 시 MDSAP 심사기관(BSI, TÜV 등) 활용 검토 → 실사 횟수 절감.

### 4-4. 불만 처리(Complaint Handling) 통합 절차 설계

ISO 13485 §8.2.1 + QMSR §820.20 + MDR Vigilance를 단일 절차서로 통합:

```
불만 접수
    │
    ▼
① 정보 기록 (ISO §8.2.1 기본)
    │
    ├─ FDA: MDR reportable 여부 판단 → 30일/5일 이내 보고 (21 CFR 803)
    ├─ EU: Serious Incident 여부 판단 → 15일 이내 NCA 보고 (MDR Art. 87)
    └─ MFDS: 중대 부작용 여부 → 15일 이내 MFDS 보고 (의료기기법 §31)
    │
    ▼
② 조사 착수 및 완료 시점 기록 (QMSR §820.20 강화 요건)
    │
    ▼
③ CAPA 연계 (ISO §8.5.2)
    │
    ▼
④ PMS 데이터 피드백 (MDR Art. 83)
```

### 4-5. 내부감사 체계 — FDA 실사 대비

QMSR 시행(2026-02-02) 이후 FDA는 내부감사 보고서를 열람할 수 있음.

- **대응 방안**: 내부감사 보고서를 ISO 13485 §8.2.2 형식으로 작성, 결과를 명확히 문서화.
- **주의**: 부적합 사항은 숨기지 말고 CAPA와 연계하여 "개선 실행" 증거로 관리.
- **NB 대응**: MDR Annex IX §3.7 — NB 불시 감사 대비, 내부감사 계획·결과 즉시 접근 가능 형태 유지.

## 6. 갭 분석 체크리스트 (현재 ISO 13485 절차서 기준 추가 필요 항목)

현행 ISO 13485 절차서 37개(#31) 대비 추가 작성이 필요한 지역별 문서:

### FDA 추가 필요
- [ ] ARR 준수 절차 (§820.10(b) — UDI, 21 CFR 803, 806 활동 통합)
- [ ] 불만 처리 절차 — MDR reportable 판단 섹션 추가
- [ ] 내부감사 보고서 양식 — FDA 열람 대비 형식 표준화

### KGMP 추가 필요
- [ ] 각 절차서 한국어 버전 (ISO 언어 정합 검토 포함)
- [ ] 전산화시스템 밸리데이션 절차서 (QMS SW 포함)
- [ ] GMP 심사 대응 체크리스트 (MFDS 빈번 지적 항목 기반)
