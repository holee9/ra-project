# IEC 62304:2006+A1:2015 SW 수명주기 산출물 매핑

## 1. 표준 개요

| 항목 | 내용 |
|---|---|
| 표준명 | IEC 62304:2006/Amd 1:2015 — Medical device software: Software life cycle processes |
| 발행기관 | IEC (국제전기기술위원회) |
| 현행 판본 | Ed 1.1 (2006 + AMD1:2015 통합본, 2015-06 발행) |
| 국내 채택 | KS P IEC 62304 (2025 확인본, KSSN) |
| EU 조화표준 | EN IEC 62304:2006+A11:2021+A1:2015 (MDR Annex I GSPR §17 소프트웨어 요건 적용) |
| FDA 인정 | Accession No. 38829, 인정일 유효 (Recognized Consensus Standard) |
| 규범 참조 | ISO 14971 (위험관리), IEC 62366-1 (사용적합성) |
| 차기 개정 | IEC 62304:2026 (DIS 단계, 안전등급 3→2 단순화 예정) |

**적용 범위:** 의료기기의 구성요소로서의 SW 및 독립형 의료기기 SW(SaMD). 하드웨어·펌웨어 포함 임베디드 SW 모두 적용.

## 2. Safety Class 판정 기준 및 자사 3개 제품 적용

### 2.1 Safety Class 정의

| Class | 정의 | 비고 |
|---|---|---|
| **A** | SW 고장이 위해 상황에 기여하지 않거나 미미 (No injury possible) | 관리 도구·로그뷰어 등 |
| **B** | SW 고장이 경미한 부상 가능 (Non-serious injury possible) | 노출 제어 보조 기능 |
| **C** | SW 고장이 사망 또는 중상 가능 (Serious injury or death possible) | 직접 방사선 제어 |

> Class 판정 → ISO 14971 위험 분석 선행 필수. FMEA/FTA로 SW 기여 hazard 식별 후 확정.

### 2.2 자사 3개 제품 Safety Class 판정

| 제품 | SW 유형 | Safety Class | 판정 근거 |
|---|---|---|---|
| **X-ray Detector** | 임베디드 FW (이미지 보정·DQE 연산) | **Class B** | 이미지 왜곡 시 오진 위험 → 경미한 부상 수준. 직접 방사선 제어 미수행 |
| **Handheld X-ray Source** | 임베디드 FW (kVp·mA·타이머 제어) | **Class C** | SW 고장 → 과노출 방사선 → 환자·술자 중상 가능 |
| **촬영실 GUI SW** | 독립형 SW (촬영 파라미터·워크플로우) | **Class C** | 촬영 파라미터 전달 오류 → 과노출 → 중상. SaMD 분류 가능 |

> **중요:** Handheld X-ray Source와 촬영실 GUI SW는 Class C이므로 IEC 62304 전 조항(Cl.5.1~5.8, 6~9) 산출물 완비 필수.

## 3. IEC 62304 조항별 산출물 매핑

### 3.1 Clause 5 — Software Development Process

#### 5.1 Software Development Planning

| 항목 | 내용 |
|---|---|
| **산출물** | Software Development Plan (SDP) |
| **필수 적용 Class** | A·B·C 전체 |
| **SDP 필수 포함 항목** | SW 안전등급 및 판정 근거 / 수명주기 모델 선택 / 적용 표준·방법론 / 각 단계 산출물 목록 / 위험관리 연계 계획 / 형상관리 계획 / 변경통제 프로세스 / 문제해결 프로세스 / 검증 전략 |
| **주의** | OTS(Off-The-Shelf) SW 및 SOUP(SW Of Unknown Provenance) 관리 계획 포함 필수 |

#### 5.2 Software Requirements Analysis

| 항목 | 내용 |
|---|---|
| **산출물** | Software Requirements Specification (SRS) |
| **필수 적용 Class** | A·B·C 전체 |
| **SRS 필수 포함 항목** | 기능 요건 / 성능 요건 / 인터페이스 요건 (HW·SW·통신) / 안전 요건 (ISO 14971 SW 위험 통제 출력값) / 보안 요건 / 사용환경 요건 / 규제 요건 추적 |
| **주의** | 요건은 테스트 가능(verifiable)하도록 정량 기술. 모호한 요건 금지 |

#### 5.3 Software Architectural Design

| 항목 | 내용 |
|---|---|
| **산출물** | Software Architecture Document (SAD) / Software Architecture Description |
| **필수 적용 Class** | B·C |
| **SAD 필수 포함 항목** | SW 시스템 구성 (SW Items 식별) / SW Units 분해 / 인터페이스 정의 / SOUP 목록 및 기능 명세 / 안전등급별 분리 설계 근거 |
| **Class A 면제** | Cl.5.3 전체 면제 가능 (단, SDP에 면제 근거 기록 필수) |

#### 5.4 Software Detailed Design

| 항목 | 내용 |
|---|---|
| **산출물** | Software Detailed Design Document (SDS) / SW Unit 사양 |
| **필수 적용 Class** | **C만 필수** |
| **SDS 필수 포함 항목** | 각 SW Unit의 입력·출력·알고리즘 상세 / 상태도·순서도 / 메모리 배치 / 인터럽트·타이밍 설계 |
| **Class A·B 면제** | Cl.5.4 전체 면제 가능 (SDP에 면제 근거 기록) |

#### 5.5 Software Unit Implementation and Verification

| 항목 | 내용 |
|---|---|
| **산출물** | Unit Test Plan / Unit Test Report / Code Review Checklist & Record |
| **필수 적용 Class** | B·C |
| **포함 항목** | Unit 단위 기능 시험 결과 / 코드 리뷰 기록 (2인 검토 권장) / 정적 분석 결과 (MISRA-C 등) / 경계값·예외 처리 시험 |
| **Class A** | Cl.5.5 Unit 검증 기록 불필요. 단, 코드 동결 시점 기록 권장 |

#### 5.6 Software Integration and Integration Testing

| 항목 | 내용 |
|---|---|
| **산출물** | Integration Test Plan / Integration Test Report / SOUP Integration Record |
| **필수 적용 Class** | B·C |
| **포함 항목** | SW Items 간 인터페이스 시험 / SOUP 포함 통합 시험 / 회귀 시험 결과 / 오류 주입(fault injection) 결과 |

#### 5.7 Software System Testing

| 항목 | 내용 |
|---|---|
| **산출물** | Software Verification Plan (SVP) / Software Verification Report (SVR) |
| **필수 적용 Class** | B·C |
| **포함 항목** | SRS 전 요건 대비 시험 (요건 추적 매트릭스 RTM 포함) / 사용 시나리오 기반 시험 / 경계조건·스트레스·성능 시험 / 이상 상황 거동 시험 |
| **RTM(Requirement Traceability Matrix)** | SRS 요건 ↔ SVP 시험 항목 ↔ SVR 결과 3방향 추적 필수 (FDA eSTAR 제출 시 핵심) |

#### 5.8 Software Release

| 항목 | 내용 |
|---|---|
| **산출물** | Software Release Record / Release Notes / Release Checklist |
| **필수 적용 Class** | A·B·C 전체 |
| **포함 항목** | 릴리즈 버전 식별자 (형상 ID) / 미해결 결함 목록 및 위험 평가 / 사용 지침·설치 절차 / 알려진 잔존 이상 (Known Anomaly List) |

### 3.2 Clause 6 — Software Maintenance Process

| 산출물 | 필수 적용 Class | 내용 |
|---|---|---|
| Software Maintenance Plan | A·B·C | 유지보수 유형 정의 (수정·적응·완전·예방) / 변경 평가 프로세스 / 재검증 범위 결정 기준 |
| Change Request Record | A·B·C | 변경 요청 → Cl.9 문제해결 프로세스와 연계 |

> A1:2015 주요 변경: SOUP 변경 관리 강화, 유지보수 중 안전등급 재평가 요구.

### 3.3 Clause 7 — Software Risk Management Process

| 산출물 | 필수 적용 Class | 내용 |
|---|---|---|
| Software Hazard Analysis | A·B·C | SW 고장 모드 → 위해 상황 연결 (FMEA/FTA) |
| Software Risk Control Measure Record | B·C | SW 요건(SRS)에 안전 요건 반영 증거 |
| Residual Risk Evaluation | B·C | SW 잔존 위험 수용 여부 판정 |
| SOUP Risk Analysis | B·C | SOUP 기여 위험 식별·평가 |

> ISO 14971과 완전 통합: Cl.7 산출물은 ISO 14971 RMF의 SW 파트로 기록.

### 3.4 Clause 8 — Software Configuration Management Process

### 3.5 Clause 9 — Software Problem Resolution Process

| 산출물 | 필수 적용 Class | 내용 |

| SW Hazard Analysis | ISO 14971 RMF (Cl.7) | SW 위험 → RMF Risk Register에 통합 |
| SRS (안전 요건) | IEC 62366-1 URRA / Use FMEA | 사용 오류로 인한 SW 위험 요건화 |
| SVP/SVR RTM | FDA eSTAR Software Section | RTM이 eSTAR 핵심 증거 |
| SCMP | MDR Annex II §6.2(d) | SW 버전 추적 = 기술문서 필수 항목 |
| SDP (사이버보안 계획) | FDA Cybersecurity Guidance (2023) + MFDS 사이버보안 가이드라인 (2025.01) | SBOM, Threat Model 연계 |
| SOUP 관리 (Cl.5.3, 8) | FDA OTS SW 가이던스 | SOUP 기능·버전·공급사 목록 필수 |

## 8. 실무 체크리스트

### 개발 착수 전
- [ ] Safety Class 판정 완료 (ISO 14971 위험 분석 선행)
- [ ] SDP 초안 작성 및 승인 (면제 조항 근거 포함)
- [ ] SOUP 목록 식별 및 위험 분석 착수

### 개발 중
- [ ] SRS → SAD → SDS(Class C) 순서 문서화
- [ ] 각 단계 검토(review) 기록 유지
- [ ] 형상 베이스라인 설정 (릴리즈 후보 기준)

### 검증·시험
- [ ] Unit Test → Integration Test → System Test 순서 수행
- [ ] RTM 완성: SRS ↔ SVP ↔ SVR 3방향 추적
- [ ] Known Anomaly List 작성 및 위험 평가

### 제출 전 최종 확인
- [ ] 모든 산출물 버전·날짜·서명 확인
- [ ] SCMP 상 산출물 형상 ID 일치 확인
- [ ] FDA: eSTAR Software Section 입력 사항 점검
- [ ] MFDS: SW 허가심사 가이드라인 체크리스트 대조
- [ ] MDR: Annex II §6.2 항목별 문서 매핑 확인
