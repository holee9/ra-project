## 2. MDCG 2020-5: 동등성 입증 (Clinical Evaluation – Equivalence)

### 2.1 핵심 개념

동등성(Equivalence)은 임상평가 과정에서 **동등 기기(equivalent device)의 임상데이터를 자사 기기의 GSPR 적합성 확인에 활용**하기 위한 전제 조건이다.  
동등성 입증은 임상평가 의무 자체를 면제하지 **않는다**. 동등 기기 데이터가 임상평가 과정에 진입할 수 있는 조건을 부여할 뿐이다.

### 2.2 동등성 3가지 특성 (MDR Annex XIV Part A (3))

MDR과 MEDDEV 2.7/1 rev.4 간 차이점 중심으로 정리한다.

#### 2.2.1 Technical Characteristics (기술적 특성)

| 평가 항목 | MDR 요건 | MEDDEV 2.7/1 rev.4 대비 차이 |
|-----------|----------|------------------------------|
| 설계 | 유사한 설계 (similar design) | 동일 수준 |
| 사용 조건 | **유사한** 조건 (similar conditions) | MEDDEV는 **동일** 조건 요구 → MDR이 완화 |
| 사양·속성 | 물리화학적 특성, 파장, **소프트웨어 알고리즘** 포함 | MDR은 SW 알고리즘 명시 추가 |
| 작동 원리 | 유사한 원리 및 핵심 성능 요건 | 동일 수준 |

> **SW 알고리즘 유사성**: 알고리즘의 코드 동일성이 아닌 **기능적 원리(functional principle)** 및 임상 성능·목적이 유사해야 함. GUI 표시 전용 SW(진단·치료 기능 無)는 동등성 평가 대상에서 제외 가능.

#### 2.2.2 Biological Characteristics (생물학적 특성)

| 평가 항목 | MDR 요건 | 주의사항 |
|-----------|----------|----------|
| 조직 접촉 소재 | **동일한** 소재·물질 (same materials) | MEDDEV의 '유사 소재 예외' 조항 **MDR에서 불허** |
| 접촉 기간·종류 | 유사한 접촉 기간 및 종류 | - |
| 용출물·분해산물 | 유사한 방출 특성 (release characteristics) | ISO 10993-18 화학적 특성 분석 활용 |

#### 2.2.3 Clinical Characteristics (임상적 특성)

| 평가 항목 | MDR 요건 |
|-----------|----------|
| 임상 상태·목적 | 동일한 임상 조건·목적, 유사한 중증도·단계 |
| 적용 부위 | **동일한** 신체 부위 |
| 대상 집단 | 유사한 연령·해부학적 구조·생리 |
| 사용자 종류 | **동일한** 사용자 유형 (전문가 vs. 일반인) — MDR 신규 추가 |
| 핵심 성능 | 유사한 임상 효과 및 특정 목적 대비 성능 |

### 2.3 동등성 입증 전제 조건

1. 3가지 특성 **모두** 충족해야 동등성 성립
2. 다수의 기기에 동등성 주장 가능하나, **각 기기별로 완전히 독립 입증** 필요
3. 서로 다른 기기의 일부 특성을 혼합하여 동등성 주장 **불가**
4. **자사 기기를 타사 기기에 동등성 주장** 시: Class III·이식형 → 상호 계약 + 기술문서 전면 접근권 필요 (Art. 61(5))
5. **동등 기기가 MDR 인증 기기이어야 함**: Class III·이식형에서 타사 기기 동등성 주장 시, 대상 기기가 MDR 인증이어야 하며 MDD/AIMDD 인증 기기는 불가

### 2.4 동등성 불가 시 유사 기기 (Similar Device) 데이터 활용 범위

동등성 입증 실패 시 유사 기기 데이터는 다음 목적에 한해 사용 가능:
- 위험 관리 파악 (hazard identification)
- State of the art 및 대안 치료 파악
- 임상평가 범위 정의
- 임상조사 설계 입력
- 임상 outcome parameter 정의

## 3. MDCG 2020-6: 레거시 기기 임상근거 요건

### 3.1 적용 대상 (Legacy Device 정의)

MDD (93/42/EEC) 또는 AIMDD (90/385/EEC) 하에 CE 인증을 받은 기기.

### 3.2 핵심 개념: "충분한 임상근거 (Sufficient Clinical Evidence)"

MDR에 별도 정의 없음. 해석: "기기가 제조사가 의도한 대로 사용될 때 안전하고 임상적 이익을 달성한다는 결론을 도출한 적격 평가의 현재 결과".  
임상평가는 **지속적 프로세스** — 1회성 이벤트가 아님.

### 3.3 레거시 기기의 MDR 임상평가 절차 (Annex XIV Part A 기반)

```
1a. Clinical Evaluation Plan 수립 / 갱신
  ↓
1b. 임상데이터 식별
  ├─ Pre-market 데이터 (기존 MDD 인허가용 임상조사, 문헌, 사례보고)
  └─ Post-market 데이터 (PMS, PMCF, 문헌, 레지스트리)
  ↓
1c. 임상데이터 평가 (Appraisal)
  └─ 방법론 품질 평가 도구 적용 (Cochrane RCT tool, MINORS, NOS 등)
  ↓
1d. 신규 임상데이터 생성 필요성 판단
  └─ 데이터 갭 분석 → PMCF 설계
  ↓
1e. 임상데이터 분석
  └─ 편익-위험 결정, State of the art 비교, 충분성 결론
```

### 3.4 임상근거 위계 (Hierarchy of Clinical Evidence)

| 순위 | 데이터 유형 | 비고 |
|------|-------------|------|
| 1 | 고품질 임상조사 결과 (전 변형·적응증 포괄) | 가장 강력 |
| 2 | 일부 갭 있는 고품질 임상조사 | 갭은 위험관리 + PMCF로 보완 |
| 3 | 레지스트리 등 고품질 임상데이터 수집 시스템 | 데이터 품질 확인 필수 |
| 4 | 방법론적 결함 있으나 데이터 정량화 가능한 연구 | 안전·성능 우려 없는 레거시 기기에서 허용 가능 |
| 5 | 동등성 데이터 (MDR 기준 충족 시) | 사후 데이터 수집으로 보완 필요 |
| 6 | State of the art / 유사 기기 데이터 | WET(Well-Established Technology)에만 GSPR 적합 지지 가능 |
| 7 | 불만·감시 데이터 | 안전 트렌드 파악용, 단독 충분성 입증 불가 |
| 8~12 | PMS 데이터, 사례보고, 비임상시험 등 | 지지적 역할 |

> **WET (Well-Established Technology) 기준**: 단순·안정 설계 + 안전 이력 + 알려진 성능 + 오랜 시장 이력 → 4가지 모두 충족 시 WET 해당 가능.

### 3.5 레거시 기기 CEP (Clinical Evaluation Plan) 최소 항목 (Appendix II)

- MDR GSPR 중 임상데이터가 필요한 요건 식별
- 기기 intended purpose 명시
- 적응증·금기사항·대상 집단 명시
- 임상 편익 세부 기술 (outcome parameters 포함)
- 대안 치료법 분석 전략
- 필요 임상근거 수준 정의 및 근거
- PMS 데이터 수집·분석 전략 (PMCF 포함)
- 레거시 기기 인증 이력 하의 PMCF 이행 여부 확인

## 4. MDCG 2020-7: PMCF Plan Template

### 4.1 PMCF 목적 (MDR Annex XIV Part B)

1. 예상 수명 전체에 걸친 안전성·성능 (임상 편익 포함) **확인**
2. 미지의 부작용 식별 및 알려진 부작용·금기 **모니터링**
3. 사실에 근거한 신규 위험 식별·분석
4. 편익-위험 비율의 지속적 허용 가능성 보장
5. 체계적 오남용 또는 off-label 사용 식별

### 4.2 PMCF Plan 필수 섹션

| 섹션 | 내용 |
|------|------|
| A | 제조사 연락처 |
| B | 기기 설명 및 사양 (intended purpose, UDI, 분류) |
| C | PMCF 활동: 방법·절차 (일반 + 특이적) |
| D | 기술문서 관련 부분 참조 (CER 및 Risk Management File) |
| E | 동등·유사 기기 임상데이터 평가 |
| F | 적용 Common Specification, 조화표준, 가이던스 |
| G | PMCF 평가보고서 예상 작성일 |

### 4.3 PMCF 활동 유형 (Section C 구성)

각 활동을 C.1, C.2 … 서브섹션으로 구분하여 기술:

| 활동 유형 | 설명 |
|-----------|------|
| Device registry | 제품별 또는 generic group 레지스트리 |
| PMCF study | 임상조사 (전향적·후향적·확장 추적 등) |
| Real-World Evidence (RWE) | 실세계 데이터 분석 (보험 DB, 병원 EMR 등) |
| Survey | HCP 또는 환자/사용자 설문 |
| Literature screening | 과학 문헌 및 임상데이터 소스 체계적 검색 |

각 활동별 필수 기술 항목:
- 활동의 필요성 출처 (NB 요청 / CER / PMS / 위험관리 등)
- 목표 (안전 확인 / 성능 확인 / 신규 부작용 식별 등)
- 절차·방법 상세
- 적절성 근거 (sample size 정당화, 비교군 선정, 편향 제어)
- 타임라인 (분기별 또는 최소 연간)

### 4.4 Section E: 동등·유사 기기 데이터 평가표

```
| 제품명 | Intended purpose | Intended users | 대상 집단 | 임상 상태 | 적응증 | CER 참조 위치 |
```
> 주의: PMCF 지속 안전·성능 데이터는 **평가 대상 기기 자체**에서 확보해야 함. 동등·유사 기기 데이터는 state of the art 갱신, 안전성 신호 파악 등 보조 목적에 한정.

## 5. MDCG 2020-8: PMCF Evaluation Report Template

### 5.1 역할

PMCF Plan에 따라 수행한 활동 결과를 문서화한 보고서.  
→ CER(임상평가보고서)의 일부이자 기술문서의 구성 요소.  
→ 결과는 CER, 위험관리 문서, PMS Plan, SSCP(해당 시) 갱신에 반영.

### 5.2 PMCF Evaluation Report 필수 섹션

| 섹션 | 내용 |
|------|------|
| A | 제조사 연락처 |
| B | 기기 설명 (변경 시 갱신, 미변경 시 Plan 참조) |
| C | 수행된 PMCF 활동 결과 (데이터 수집 결과, 긍정·부정 포함, 편차 근거) |
| D | 동등·유사 기기 임상데이터 평가 결과 |
| E | 기술문서에 대한 결과의 영향 (CER, Risk Management File, SSCP 각각 명시) |
| F | 적용된 CS, 조화표준, 가이던스 |
| G | 종합 결론 (PMCF 목적 대비 결과, 예방·시정 조치 필요성) |

### 5.3 Section E: 기술문서 영향 분석 요건

1. **CER 영향**: 관련 정보 분석 → 갱신된 CER에 반영
2. **Risk Management File 영향**: 신규 위험 또는 기존 위험 재평가 결과 반영
3. **(해당 시) SSCP 영향**: Summary of Safety and Clinical Performance 갱신 여부

| 2020-5 (동등성) | Predicate (동등 기기) 있으면 활용 가능. SW 알고리즘(이미지 재구성, 보정 알고리즘) 유사성 입증 필요 | 동등성 비교표 작성 — SW 알고리즘 항목 특히 세밀히 기술 |
| 2020-6 (레거시) | 신규 기기이면 해당 없음. 기존 기기를 MDR 전환 시 WET 해당 가능 여부 검토 | MDD 인증 이력 있으면 CEP 갱신, PMCF 이행 이력 확인 |

## 8. 제출 체크리스트

### 8.1 CE 마킹 Technical Documentation (MDCG 2020-5/6 관련)

- [ ] Clinical Evaluation Plan (CEP) 작성 — Annex XIV Part A 기반
- [ ] 동등 기기 목록 확정 및 동등성 비교표(Annex I) 작성
- [ ] 동등 기기 기술문서 접근 계약 체결 (타사 + Class III·이식형)
- [ ] 동등 기기가 MDR 인증 여부 확인 (Class III·이식형: MDR 인증 필수)
- [ ] 임상근거 위계 분석 및 데이터 갭 목록 작성
- [ ] Clinical Evaluation Report (CER) 작성 — MDCG 2020-13 템플릿 활용
- [ ] GSPR 매트릭스에 임상데이터 매핑

### 8.2 PMCF Plan (MDCG 2020-7 관련)

- [ ] Section A~G 전 항목 작성
- [ ] 각 활동별 sample size 통계 근거 명시
- [ ] 타임라인 분기별/연간 정의
- [ ] PMCF Report 예상 발행일 명시 (Section G)

### 8.3 PMCF Evaluation Report (MDCG 2020-8 관련)

- [ ] 수행 활동 결과 전부 기술 (긍정·부정 포함, 편차 근거)
- [ ] Section E: CER·RMF·SSCP 영향 분석 기재
- [ ] Section G: 종합 결론 + 다음 PMCF Plan 입력 사항 기재
- [ ] CER 갱신본에 PMCF Report 참조 포함
