# 3개 지역 공통 빈번 지적사항 Top 20

의료기기 인허가 심사(MFDS·FDA·MDR)에서 반복적으로 발생하는 지적사항을 공식 데이터 및 Notified Body 가이던스 기반으로 정리. 자사 3개 제품(X-ray Detector · Handheld X-ray Source · 촬영실 GUI SW)에 대한 영향 맵핑 포함.

## 1. 통계 개요

| 지역 | 주요 지표 | 출처 |
|------|-----------|------|
| FDA 510(k) | AI request 발생률 ~67%, RTA hold 발생률 ~33%, NSE 결정률 ~15% | FDA/MedDeviceGuide 2025-2026 |
| MDR NB | 기술문서 검토 지연 주요 원인 3개: 정보 불충분·구조 미흡·과도한 범위 | Team NB BPG V3 2025-04 |

## 3. 지적사항별 상세 분석

### #1 기술문서 구조·완성도 미흡

**사실 (Fact)**
- FDA: RTA hold 발생률 ~33%. eSTAR 의무화(2023-10) 이후 감소 추세이나 여전히 최다 지연 원인.
- MDR: Team NB BPG V3(2025-04) 명시 — 문서 구조 불명확, 정보 검색 불가로 검토 지연.
- MFDS: 기술문서 작성 기준(고시 제2024-88호 별표 1~3) 미준수 시 즉시 보완 요청.

**해석 (Interpretation)**
RTA/NB 접수 단계에서 걸러지는 가장 예방 가능한 결함. 완성된 구조가 없으면 내용 심사 자체가 진행되지 않음.

**행동 지시 (Action)**
- FDA: eSTAR 전 항목에 N/A 포함 기재 완료 여부 확인.
- MDR: Annex II/III 구조로 TD 목차 일치 확인, 검색 가능한 링크/참조 구조 유지.
- MFDS: 기술문서 항목별 체크리스트(의약품의료기기법 시행규칙 별지) 최신 버전 사용.

---

### #2 성능시험 데이터 불충분·방법론 결함

**사실 (Fact)**
- FDA: 510(k) NSE의 가장 빈번한 실질적 원인. Bench test 방법론·Pass Criteria 미설정, 통계적 유의성 부재.
- MDR: Team NB — "완전한 최신 시험보고서 요구. 약식·부분 보고서 불인정."
- MFDS: 성능시험 자료 미비(시험기관 인정 현황·시험 방법 불명확) 보완 요청 최다.

**해석 (Interpretation)**
시험은 수행했으나 보고서에 방법론·조건·결과·판정 기준이 누락되는 경우가 다수.

**행동 지시 (Action)**
- 모든 성능 항목에 대해 ISO/IEC 방법론 명시, PASS/FAIL 기준 사전 설정.
- 인정기관(KOLAS/A2LA/ILAC) 시험 성적서 첨부.
- eSTAR Section 8(Performance Testing Summary) 또는 MFDS 성능시험 자료 항목별 기재.

---

### #3 위험관리 파일(ISO 14971) 미흡

**사실 (Fact)**
- 3지역 공통: ISO 14971:2019 적용 의무. 위험 식별·추정·평가·저감·잔존위험 수용 전체 과정 문서화 필요.
- FDA: Risk analysis 결함 → AI request 및 NSE 원인 중 주요 비중.
- MDR: GSPR §1 요구: risk management system 전 주기 문서 → 단순 FMEA 테이블만으로 불충분.

**행동 지시 (Action)**
- Risk Management Plan / Risk Assessment / Risk Management Report 3문서 구분 유지.
- SW, 사이버보안, 방사선 위험 포함 망라적 Hazard 목록 작성.
- Hazard → Harm 인과 경로(causal chain) 명시, 잔존위험 수용 기준 근거 제시.

---

### #4 라벨링·IFU 기재 결함

**사실 (Fact)**
- FDA 21 CFR 801: Indications for Use(Form 3881)와 510(k) 내 IFU 내용 불일치 → AI request.
- MDR §23.4: 36개 의무 기재 항목. 누락 시 NB deficiency.
- MFDS 고시 제2024-77호: 디지털의료기기 추가 기재 사항(SW 버전·사이버보안·데이터 형식 등).

**행동 지시 (Action)**
- IFU 기재 항목을 3지역 체크리스트(`IFU_필수요소_3지역_비교.md`)와 교차 검토.
- 라벨·IFU·MFDS 신청서·eSTAR/TD 간 사용목적·금기사항·경고문 일치 여부 확인.

---

### #5 사용목적·적응증 기재 불명확·불일치

**사실 (Fact)**
- FDA: Intended Use와 Indications for Use 혼용, 과도하게 광범위한 기재 → NSE 위험.
- MDR: Intended purpose (Art. 2(12)) 문서 전체 일관성 요구.
- MFDS: 품목 허가증·기술문서·IFU 간 사용목적 불일치 → 즉시 보완.

**행동 지시 (Action)**
- Intended Use(사용목적) Master 문서 1개 지정 → 모든 문서에 그대로 복사 붙여넣기.
- 사용목적 변경 이력 관리(설계변경 절차서 연동).

---

### #6 소프트웨어 문서화 결함(IEC 62304)

**사실 (Fact)**
- FDA: Software Level of Concern 판단 오류, SDS·SRS·SVP 부재, V&V gap.
- MDR: IEC 62304 적용 시 Safety Class B/C 산출물 전체 요구.
- MFDS: SW 기술문서(소프트웨어 버전·SOUP 목록·테스트 절차) 미비.

**행동 지시 (Action)**
- GUI SW: Class C 수준(생명에 직결) → 전체 산출물 유지 필수.
- Detector SW 및 Handheld Control FW: Class B 수준 적용.
- SOUP 목록(버전·기원) 및 문제 해결 기록 유지.

---

### #7 임상평가(CER/SE 동등성) 근거 미약

**사실 (Fact)**
- FDA: 동등 기기(predicate)와 기술적 차이 설명 없이 동등성 주장 → NSE.
- MDR: MDCG 2020-5 동등성 기준(기술·생물·임상 3항목) 전부 충족 필요.
- MFDS: 임상적 동등성 확보 불가 시 임상시험 자료 제출 요구.

**행동 지시 (Action)**
- CER(EU)/SE Matrix(FDA)/임상자료(MFDS) 별도 전략 수립.
- Predicate 또는 동등 기기 선정 이유 문서화(SOP 수립).
- 최신 문헌 검색 전략(PubMed, MAUDE, EUDAMED) 기록 유지.

---

### #8 생체적합성 평가 결함(ISO 10993-1)

**사실 (Fact)**
- FDA: 접촉 기간·조직 유형 오분류, 시험 누락 → AI request 빈발.
- MDR: ISO 10993-1:2018 기반 전략 문서(Biological Evaluation Plan/Report) 요구.
- MFDS: 생체적합성 시험성적서 또는 면제 근거 미비.

**행동 지시 (Action)**
- X-ray Detector: 간접접촉(환자·사용자 접촉 없음) → 면제 근거 문서화.
- Handheld Source: 사용자 손 접촉(surface contact, limited) → 세포독성 등 최소 시험.
- 면제 주장 시: 기존 데이터·재료 인증서·동일 재료 이전 사용 이력 근거 작성.

---

### #9 문서 간 정합성 부족

**사실 (Fact)**
- Team NB BPG: "UDI, intended use, contraindications, warnings 등 중복 기재 항목의 버전 불일치"를 명시적 지적 사유로 열거.
- FDA: Form 3881과 510(k) labeling section 내용 불일치 → RTA hold.

**행동 지시 (Action)**
- 문서 관리 시스템(DMS) 내 마스터 데이터 단일화.
- 제출 전 Cross-reference Matrix 작성(문서별 UDI·사용목적·경고문 일치 확인).

---

### #10 사이버보안 문서 미비

**사실 (Fact)**
- FDA: Section 524B(FD&C Act) — 모든 Cyber Device에 SBOM 의무화(2023-03 시행). 2026-02 QMSR 개정으로 사이버보안 QMS 통합 강화.
- MDR: MDCG 2019-16 Rev.1 — 사이버보안 위험관리 요구. GSPR §17.2 연결.
- MFDS: 2025-01 개정 사이버보안 가이드라인 — 요구사항 15→35개 확장(RA-01~RA-35).

**행동 지시 (Action)**
- GUI SW·Handheld Source: Cyber Device 해당 여부 판정 → SBOM 생성.
- Threat Model(STRIDE 기반) 작성, Penetration Test 결과 첨부.
- MFDS RA-01~RA-35 체크리스트(`MFDS_사이버보안_가이드라인_2025_대응.md`) 참조.

---

### #11 Predicate/동등 기기 선정 오류

**사실 (Fact)**
- FDA: 재심 사례 분석 → recalled predicate 사용, 다른 Intended Use predicate 선정이 NSE의 주요 원인.
- MDR: MDD 기반 동등 기기 → MDR 시장출시 제품으로 교체 필요(전환기간 이후).

**행동 지시 (Action)**
- 510(k) DB(accessdata.fda.gov) 검색 시 recall 상태 확인.
- Pre-Submission(Q-Sub)을 통해 predicate 선정 FDA 사전 확인.

---

### #12 EMC·전기안전 시험 미흡

**사실 (Fact)**
- 3지역 공통: IEC 60601-1 Ed3.2 + IEC 60601-1-2:2014+A1:2020 시험 의무.
- 빈번 지적: 시험 조건 부적절(최악 조건 미적용), Pass Criteria 미설정, 발행연도 구버전 표준 사용.

**행동 지시 (Action)**
- IEC 60601-1-2 기반 필수·권고 시험 항목 매트릭스(`IEC60601-1_Ed3.2_시험항목_매트릭스.md`) 참조.
- 시험 표준 버전·발효일 확인 (3지역별 인정 현황 상이).

---

### #13 방사선 안전성 서류 미흡 (X-ray 특화)

**사실 (Fact)**
- FDA: 21 CFR 1020.30~1020.33 적합성 — 방사선 방출 보고서(Radiation Emission Report) 필수.
- MFDS: 방사선 안전 기준 적합 자료 미비 빈발.
- MDR: IEC 60601-1-3:2008+AMD1+AMD2 준수 — 누설 방사선·HVL·콜리메이터 시험.

**행동 지시 (Action)**
- Handheld Source: NCRP Report 102 허용 선량율(1mGy/h @ 5cm) 근거 측정 자료 준비.
- Detector: 간접 방사선(scatter) 차폐 구조 검증 자료.
- 시험 성적서에 시험 조건(kVp, mA, 거리) 명시.

---

### #14 인간공학(Usability) 파일 미비

**사실 (Fact)**
- 3지역: IEC 62366-1:2015+AMD1:2020 적용. Use Error → 환자 위해 연결 시 심각한 지적 사유.
- FDA: Human Factors 전용 guidance(2026-05 최신) — HF Study 결과·Use-related Hazard 분석 요구.

**행동 지시 (Action)**
- GUI SW: 촬영실 인터페이스 오류 → 과피폭 위험. Class C 수준 Usability 파일 유지.
- Summative Evaluation Study 결과 제출 전략 수립.

---

### #15 GSPR 준수 입증 불충분 (MDR 특화)

**사실 (Fact)**
- Team NB BPG(2025): "GSPR 섹션당 시험보고서 단순 나열은 불충분. 위험관리(ISO 14971)와 연동된 GSPR 준수 논거 필요."
- MDR Annex I §1~§23 전 항목 입증 요구.

**행동 지시 (Action)**
- GSPR 체크리스트(`MDR_AnnexI_GSPR_Checklist.md`)에 각 항목별 "입증 방법"과 "참조 문서" 기재.
- 단순 "적용됨/미적용" 체크 방식 지양.

---

### #16 PMS/PMCF 계획 부실 (MDR 특화)

**사실 (Fact)**
- MDR Annex III: PMS Plan·PMCF Plan·PSUR(Class IIa~III) 별도 문서 요구.
- NB 지적: PMCF 활동 항목 구체성 결여("일반적인 문헌 검색"만 기재 → 불충분).

**행동 지시 (Action)**
- PMCF Survey·Registry Study·Literature Surveillance 구체 일정 및 방법 기재.
- Class IIa X-ray Detector: 연간 PSUR 제출 대비.

---

### #17 설계 변경 관리 기록 부재

**사실 (Fact)**
- FDA/MFDS/MDR 공통: 설계 이력(DHF/Design History) 내 변경 이력 불비 시 심사 지연.
- DQS NB 지적: "변경 후 수정된 성적서만 제출, 이전 버전 대비 변경 사유 미기재."

**행동 지시 (Action)**
- 설계변경 이력 문서(Design Change History) 버전별 유지.
- 변경 → 재시험 범위 판단 근거(위험관리 관점) 문서화.

---

### #18 GMP/QMS 심사 준비 미흡

**사실 (Fact)**
- MFDS: GMP 적합인정심사 + 기술문서 심사 동시 진행. 절차서 부재 또는 미이행 기록 발견 시 보완.
- FDA QMSR(2026-02-02 시행): ISO 13485:2016 기반 — ARR, 불만 파일, 내부감사 강화.
- MDR: ISO 13485 인증 + MDSAP 활용 가능.

**행동 지시 (Action)**
- QMS 절차서 목록·이행 기록 점검(`ISO13485_2016_절차서_목록_뼈대.md` 참조).
- 내부심사 주기 준수(연 1회 이상), CAPA 기록 최신화.

### #19 보완자료 불완전 답변·기한 미준수

**사실 (Fact)**
- FDA: AI Request → 180일 내 응답 필수. 미응답 시 자동 철회.
- MFDS: 보완 60일(IND 2차 30일) 초과 시 신청 무효. 불완전 답변 재보완 발생 시 기간 추가 소요.
- MDR: NB clock-stop 최대 4회/회당 90일. 반복 지적 시 신뢰도 저하.

**행동 지시 (Action)**
- 보완 접수 즉시 Tracker 등록(마감일·담당자·문서 목록).
- 지역별 대응 전략 문서 참조: `FDA_AI_Additional_Information_대응_전략.md`, `MFDS_보완자료_대응_전략.md`, NB Deficiency Letter 대응 전략(Issue #60).

### #20 제출 양식 오류·구버전 양식 사용

**사실 (Fact)**
- FDA: 구버전 Form 3601/3514/3381 사용 → 즉시 RTA hold. eSTAR 미사용(510(k) 2023-10 이후 의무).
- MFDS: 최신 별지 서식 미사용(의약품의료기기법 시행규칙 별지 30호~32호) → 보완.

**행동 지시 (Action)**
- 제출 전: FDA eSTAR 최신 버전(FDA.gov 다운로드) + MFDS 전자민원창구 최신 양식 확인.
- 양식 업데이트 알림 RSS/공지 구독.
